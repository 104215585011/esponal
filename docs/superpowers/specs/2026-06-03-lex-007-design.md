## 0. 一句话目标

把现有的"幼稚回填"——每次 AI（DashScope）查词结果直写 `LexiconEntry`、与精库同表、无质量闸、下次即命中——升级为**带分层 + 本地置信度闸 + 高频缺口计数**的管线，在不增加任何额外模型调用的前提下**削减回落成本**并保护金库不被污染。

## 1. 现状（代码事实）

查词主流程 `src/app/api/vocab/lookup/route.ts` → `GET`：

1. `findLexiconLookupEntry(word)`（`src/lib/lexicon.ts`）读 `LexiconEntry`，命中即返回（免 AI）。
2. 未命中 → `lookupDictionary(word)`（`src/lib/dictionary.ts`）：查 `extension/lemma-dict.json` + Redis 缓存 + DashScope AI + 规则变位 `tryConjugateVerb`，返回给用户。
3. `scheduleLexiconBackfill(entry)`（route.ts 内）在 `setTimeout(0)` 里把 AI 结果 `upsertLexiconEntry`，写死 `licenseCode: "external-lookup"`、`sources: ["external-lookup"]`、`qualityScore: 1`。

`LexiconEntry`（`prisma/schema.prisma`）已有字段：`kind` `lemma` `partOfSpeech` `examples`(Json) `morphology`(Json) `sources`(String[]) `licenseCode` `qualityScore`(Int) `lookupCount`(Int) 等，`@@unique([kind, lemma])`，已有 `@@index([lookupCount])`，已有 `incrementLookupCount(id)`。

**问题**：①AI 挖的词与 license-clean 种子词混在同一张表、无标记，违反 epic"金库不被 AI 直接写"；②无置信度、无质量闸，低置信错答案会被当成命中长期服务；③无"缺口词频次"概念，无法让高频词优先升级/复审。

## 2. 设计决策（已与用户确认）

| # | 决策 | 取舍理由 |
|---|---|---|
| D1 | **同表加 `status` 字段**分层，不另建候选表 | 改动最小：一个 migration + 几处查询改 `where`。避免双表查询/升级搬数据的复杂度。 |
| D2 | **本地信号打分**，零额外模型调用 | 再调第二个模型会翻倍该词成本，与"削减回落成本"目标直接冲突。MVP 用确定性本地信号即可分流。 |
| D3 | **缺口频次复用 `lookupCount`** | 已有字段 + 索引 + `incrementLookupCount`，不加表、不加写入路径。 |

## 3. 数据模型

```prisma
enum LexiconStatus {
  vault      // license-clean 种子/人工核准，可服务，AI 不得覆写
  candidate  // AI 挖掘 + 本地高置信，可服务（免 AI），待人工升级
  review     // AI 挖掘 + 本地低置信，不服务，入待审队列
  rejected   // 人工拒绝，永不服务、AI 不得覆写
}

model LexiconEntry {
  // ... 现有字段不变 ...
  status        LexiconStatus @default(vault)
  // ...
  @@index([status])
}
```

**Migration 数据回填**（一次性）：
- 现有 `licenseCode = "external-lookup"` 的行（旧幼稚回填）→ `status = candidate`。
- 其余行（license-clean 种子）→ `status = vault`。
- 用 `@default(vault)` 保证旧迁移/种子脚本插入的行默认进金库。

## 4. 查词读取优先级

`findLexiconLookupEntry(word)` 改为只服务 `status ∈ {vault, candidate}`：

- **命中 vault/candidate** → 返回，免 AI（**省钱点**）。vault 优先于 candidate（同 lemma 不会并存，因 `@@unique([kind, lemma])`，故实际是单条；status 过滤即可）。
- **命中 review/rejected** → 视为"未命中"，落到 `lookupDictionary`（AI）。但若该 lemma 已存在 `review` 条目，则 `lookupCount++`，让高频低置信词在审核队列里冒头。
  - 实现：新增 `findReviewEntryForBump(word)` 或在 `findLexiconLookupEntry` 返回时区分（见计划）。

## 5. 本地置信度打分（零调用）

新函数 `scoreLexiconEntry(input): { score: number; reasons: string[] }`，满分 100，阈值 **T = 60**（经验值，实现时可校准）：

| 信号 | 加/减分 |
|---|---|
| lemma 命中 `lemma-dict.json`（非推断 `inferLemma`） | +25 |
| 动词：`tryConjugateVerb(lemma)` 成功 / 名词形通过 `validateNounForms` / 形容词形通过 `validateAdjectiveForms` | +25 |
| 有 ≥1 条真例句（`es` 和 `zh` 均非空） | +20 |
| 释义 ≥1 且无占位符（`?` / U+FFFD 替换符） | +20 |
| 有 `partOfSpeech` | +10 |
| `degraded === true` 或任一字段含占位符 | 直接判 `review`（封顶到阈值以下） |

- `score ≥ 60` → `candidate`（免费服务）；否则 → `review`（入队、不服务）。
- **确定性字段仍由规则引擎产出**：变位/性数走 `tryConjugateVerb` / `validateNounForms` / `validateAdjectiveForms`，AI 只供软字段（中文释义/用法）。落实 LEX-005 假变位教训。

## 6. 回填重写

`scheduleLexiconBackfill(entry)` 改为置信度感知：

1. 调 `scoreLexiconEntry` 得 `score` 与目标 `status`（candidate/review）。
2. `upsertLexiconEntry({ ..., status, qualityScore: score })`。
3. **硬约束（守卫）**：`upsertLexiconEntry` 内，当已存在条目且其 `status ∈ {vault, rejected}` 时，**只 `incrementLookupCount`，绝不覆写**该条目的内容/状态。仅 `candidate`/`review` 条目可被自动回填更新（且 review→candidate 升级允许，candidate→review 降级不允许，避免抖动）。

## 7. 审核队列（仅 stub）

LEX-007 只产出数据 + 一个只读 helper：

```ts
listReviewQueue(limit = 50): Promise<LexiconEntry[]>  // status=review，按 lookupCount desc
```

真正的 admin 审核 UI、升级金库、用户纠错入口属 **LEX-008**，本票不做。

## 9. 测试策略（TDD）

`tests/lex007.test.mjs`，红先于实现：

1. **打分**：高信号条目（lemma 命中 + 变位成功 + 真例句 + 干净释义）→ score ≥ 60 → `candidate`；`degraded`/占位符条目 → `review`。
2. **读取过滤**：`findLexiconLookupEntry` 服务 vault 与 candidate，跳过 review/rejected。
3. **守卫**：自动回填遇 `vault` / `rejected` 条目不覆写内容/状态，只 bump count。
4. **review-miss 计数**：命中 review 条目时 `lookupCount` 自增。
5. **schema 契约**：`status` 字段 + 默认 vault + 枚举值齐全（沿用项目已有 source-contract 测试风格，对 schema.prisma 做断言）。
6. 回归切片 + `npm test` 全绿 + `npx tsc --noEmit` + `npm run lint:encoding` + `npm run build`。

## 10. 价值与验证

- **省钱**：缺口词第二次起从 candidate 本地命中，免 DashScope，直接对接积分制毛利。
- **金库安全**：AI 挖的词进 candidate 层、带标记，vault/rejected 永不被自动覆写。
- **为 LEX-008 铺路**：review 队列 + lookupCount 排序，高频低置信词优先复审。
