# Esponal 产品设计文档

**日期**：2026-05-13
**版本**：v1.0
**目标用户**：中文母语者，西班牙语零基础（A1-A2）

---

## 一、产品定位

Esponal 不是课程平台，是**西语学习伴侣**。

核心理念：语言习得发生在真实、有趣的输入中，而不是枯燥的机械练习。用户不需要每日任务、连续打卡、XP 积分——他们只需要消费自己感兴趣的西语内容，系统在旁边静静积累。

**与 Duolingo 的本质差异**：
- Duolingo 目标是留存（retention），Esponal 目标是习得（acquisition）
- Duolingo 是义务，Esponal 是兴趣驱动
- Duolingo 只有识别练习，Esponal 建立真实语境下的个人语料库

**与 Language Reactor / LingQ 的差异**：
- 专为中文母语者设计，语法解释全部用中文写，针对中文没有的概念专项处理
- 西语词形还原：`fui / fueron / vas` 全部归入 `ir`，竞品做不到
- 课程词汇与真实内容消费打通，共享一套词库

## 三、课程侧——"入门启动器"

### 设计原则

课程不是主菜，是启动器。目标是帮零基础用户在 4-6 周内建立足够底子，然后能开始消费真实的西语内容。没有强制顺序，没有每日任务，用户自由节奏推进。

### 三阶段结构

**阶段一：发音 + 生存词汇（约 1 周）**
- 西语发音规则，用中文类比解释（例："j" 不是英语的 j，是类似汉语"喝"的声母）
- 最高频 300 词（名词/动词/形容词各约 100）
- 以句子为单位呈现，不是词表；配 TTS 音频

**阶段二：基础句型 + 核心语法（约 2-3 周）**
- 现在时动词变位（优先：ser / estar / tener / ir / querer / poder）
- 阴阳性规则 → 专为中文母语者写的解释（中文无此概念，需专项训练）
- 名词性数一致
- 基础句型：我是 / 我有 / 我要 / 我喜欢 / 我去...
- AI 生成情境对话（真实场景：咖啡馆、问路、自我介绍）+ TTS 固化为音频文件

**阶段三：出发（约 1-2 周）**
- 推荐入门友好的 YouTube 频道（慢速西语、手势辅助类）
- 引导用户安装 Chrome 插件，开始消费真实内容

### 内容生产方式（分层质量控制）

| 内容类型 | 生产方式 | 质量保障 |
|---|---|---|
| 语法解释（变位规则、阴阳性）| 人工整理，写入知识库 | 一次写好，长期复用 |
| 情境对话脚本 | MiniMax M2.7 生成 + 人工审核 | 审核通过后 TTS 固化，不再动态生成 |
| 练习题（填空、变位、中译西）| MiniMax M2.5 按规则批量生成 | 规则确定，答案可程序验证 |
| TTS 音频 | Azure Cognitive Services（西语 Neural TTS）| 神经网络 TTS，音质自然 |

**关键原则**：对话脚本和音频文件一旦生成并审核，即固化存储，不实时调用 AI。

### 语法知识库

中文母语者的专项痛点，每条都是独立知识单元：

- 动词变位表（所有规则动词 + 20 个最高频不规则动词）
- 阴阳性判断规则 + 常见例外
- 名词单复数变化
- ser vs estar 的区别（中文母语者最常犯的错）
- 过去时：简单过去 vs 过去未完成（另一大难点）

知识库条目与词汇库关联——用户查某个词时，可以一键看它涉及哪条语法规则。

## 四、Chrome 插件——沉浸消费

### 设计原则

插件不打断用户，不弹出复习要求，不显示进度压力。它只是让用户在消费感兴趣内容的同时，把每次遭遇变成积累。

### 核心功能

**① 双语字幕叠加**
- 自动提取 YouTube 字幕（官方字幕优先，降级到自动生成字幕）
- 西语原文 + 中文翻译同时显示，可切换显示/隐藏中文
- 字幕逐句高亮，跟随视频进度

**② 词形还原 + 点击查词**
- 字幕中每个词可点击
- 点击后弹出卡片：
  - 词根（如 `fui` → `ir`）
  - 词性 + 当前变位形式说明（简单过去时 / 第一人称单数）
  - 中文释义
  - 来自语法知识库的相关规则链接
  - 按钮：[加入我的词库] [查看所有变位]
- 词形还原引擎：优先使用 spacy 西语模型（服务端），降级使用 JS 规则库（compromise）

**③ 已学词高亮**
- 课程里学过的词 → 绿色高亮
- 插件里已保存过的词 → 蓝色高亮
- 完全陌生的词 → 不标注（不干扰阅读）

**④ 个人语料库写入**
- 用户点击"加入我的词库"后，系统记录：
  - 词根
  - 原始变位形式
  - 视频 URL + 时间戳
  - 完整原句
  - 中文翻译
  - 保存时间

### 刻意不做的功能

- 不强制弹出复习提醒
- 不打断视频要求做题
- 不显示"今天学了多少词"的进度条
- 不要求每日使用

## 五、个人语料库——核心数据模型

### 词条结构

```
词条：ir（去）
├─ 状态：学习中（new / learning / known）
├─ 遭遇记录：
│   ├─ [YouTube] "Un día en Madrid" 03:42 → "Ayer fui al mercado"
│   ├─ [YouTube] "Cocina española" 11:20 → "Fueron a comprar ingredientes"
│   └─ [课程] 阶段二 · 第3课 → "¿Adónde vas?"
├─ 见过的变位形态：fui / fueron / vas / ir / iba
├─ FSRS 复习数据：stability / difficulty / due_date
└─ 用户笔记（选填）
```

### 用户在 Web 端的词库视图

- 按词根浏览所有词条
- 点击任意遭遇记录 → 跳回 YouTube 原视频该时间戳
- 可手动标记词的掌握状态
- 可选择性触发 FSRS 复习（不强制）

## 六、技术选型

| 模块 | 选型 | 说明 |
|---|---|---|
| Web 框架 | Next.js 14 App Router | 现有脚手架 |
| 数据库 | PostgreSQL + Prisma | 现有脚手架 |
| 缓存 | Redis | 字幕缓存、会话 |
| 认证 | NextAuth.js | 现有脚手架 |
| 西语词法分析 | spacy（Python 服务，es_core_news_sm 模型）| 词形还原主引擎 |
| TTS 音频 | Azure Cognitive Services Neural TTS | 西语音质好，生成后静态存储 |
| AI 内容生成 | MiniMax M2.5/M2.7（via aicodee.com API，OpenAI 兼容格式）| M2.7 生成对话，M2.5 生成练习题 |
| Chrome 插件 | Manifest V3 + React | 与 Web 端共享类型定义 |
| YouTube 字幕 | YouTube Data API v3 | 官方字幕提取，无字幕降级自动字幕 |
| SRS 算法 | FSRS（ts-fsrs 库）| 比 SM-2 效率高 20-30% |

### 技术风险

**YouTube 字幕质量**：自动生成字幕准确率约 85-90%，用户体验与 YouTube 本身一致，可接受。

**词形还原准确率**：spacy es 模型对常见词准确率 >95%，生僻词/俚语降级处理（作为独立词条），不影响核心体验。

**AI 内容质量**：通过"生成后固化"策略解决——AI 生成的对话脚本经人工审核后才进入课程，审核通过后不再重新生成。

## 七、开发优先级

```
第一期（MVP）：Web 端入门包 + 个人词库基础
  ├─ 阶段一课程内容（发音 + 300 高频词）
  ├─ 语法知识库框架
  ├─ 词库数据模型 + 基础 UI
  └─ 用户认证（已有）

第二期：Chrome 插件 MVP
  ├─ YouTube 字幕提取 + 双语叠加
  ├─ 词形还原引擎集成
  ├─ 点击查词 + 加入词库
  └─ 已学词高亮

第三期：打通 + 深化
  ├─ 课程词汇 ↔ 插件词库统一
  ├─ FSRS 复习队列（可选触发）
  ├─ 遭遇记录跳回视频功能
  └─ 阶段二、三课程内容
```
