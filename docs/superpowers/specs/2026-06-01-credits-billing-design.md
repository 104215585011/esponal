# Esponal 积分付费制度 — 设计 spec

**日期**：2026-06-01
**作者**：Claude1（PM）+ 用户
**状态**：待用户审阅 → 转实现计划
**参考**：DejaVocab 计费体系（商业包装借鉴），适配 Esponal 真实架构与成本

---

## 1. 背景与目标

Esponal 给用户使用会产生真金白银的可变成本（字幕抓取、AI 翻译、AI 对话、TTS、查词回落等付费 API）。需要一套**积分（配额）付费制度**对成本进行变现与控制。

**核心策略（用户已定）**：
- **大量免费用户作为基本盘** + **轻量订阅付费**。
- 目标用户：成人 / 学生自学者（不做儿童早教）。
- 商业包装借鉴 DejaVocab（验证过的成熟方案），但**配额逻辑按 Esponal 自己的成本结构设计**——不照搬"字幕全免费"的假设。

**与 DejaVocab 的本质差异（设计前提）**：
- DejaVocab 是**浏览器扩展**,寄生在 YouTube/B站 页面,客户端白抓原生字幕 → 抓字幕≈0 成本。
- Esponal 是**独立目的地网站**,iframe 嵌入视频跨域拿不到字幕 → 走 **Supadata（付费）抓取 + DashScope（付费）翻译**。
- 故 Esponal 配额必须覆盖真实 AI 成本,且用**缓存命中**机制摊薄共享成本。

## 2. 等级体系

### 2.1 订阅等级（价格已定，对标 DejaVocab 本地化为人民币）

| 等级 | 月付 | 年付（省 16%） | 月度配额 | 试用 |
|---|---|---|---|---|
| **基础版** | ¥0 永久免费 | ¥0 | 注册一次性 **50**（不补充） | — |
| **进阶版** ⭐推荐 | ¥38/月 | ¥365/年（≈¥30.4/月） | **500/月** | 3 天免费 |
| **高阶版** | ¥48/月 | ¥458/年（≈¥38.2/月） | **1000/月** | 3 天免费 |

### 2.2 共建者（终身买断，限量饥饿营销）

| 等级 | 价格 | 月度配额 | 特点 |
|---|---|---|---|
| **终身进阶** | ¥1498 一次性 | 500/月**累加结转** | 含进阶全部 · 专属会员编号 · 和创始人交友 · **限量 500 名（进度条）** |
| **终身高阶** ⭐ | ¥1998 一次性 | 1000/月**累加结转** | 含高阶全部 · 新功能优先 · 同上社区权益 · 限量 500 名 |

### 2.3 配额（积分）账户机制

三类配额，行为不同：

| 类型 | 来源 | 刷新规则 | 结转 |
|---|---|---|---|
| 免费配额 | 注册赠送 50 | **一次性，不补充** | 否 |
| 订阅配额 | 进阶/高阶月度发放 | **每月重置为额度（覆盖式）** | **否**（上月剩余作废） |
| 终身配额 | 共建者月度发放 | **每月发放并累加** | **是**（永久结转） |

- 用户同一时刻只处于一个等级；余额 = 该等级当前配额。
- **存储**：配额以整数小单位存储（×100，"分配额"），消除小数精度问题（0.02→2、0.05→5、0.1→10、0.5→50）。
- **升级**：月度订阅可随时升级（立即生效，配额按新等级处理）；**不支持降级**。
- **停止续订**：到期掉回免费层；**订阅配额清零**；不再补发免费 50（注册一次性已发过）。终身累加配额不受影响。

## 3. 配额消耗逻辑（核心）

**原则：按"真实 AI 成本 / 净新增计算量"收费，按"消费已有内容"免费。**

### 3.1 消耗配额的动作

| 动作 | 建议消耗（**待按真实 API 成本标定**） | 说明 |
|---|---|---|
| **网站内全新视频字幕解锁**（Supadata 抓取 + DashScope 翻译，**仅缓存未命中**） | 一口价按时长档：≤10min=2 / 10–30min=5 / >30min=10 | ⭐ 解决 Supadata/翻译成本，且防刷爆预算。一口价用户可预估（已否决按句计）。**通过 Chrome 插件进入则免费**——见 §3.3 |
| 短语/搭配高亮（PHRASE-001 + LEX-003，进阶+ 专属） | 0.05/句 | 真实现有功能；免费层无此功能 |
| AI 对话（talk，每轮） | 0.5/轮 | Esponal 特有，DejaVocab 无 |
| TTS 发音（每次） | 0.1/次 | 可缓存，按未命中计 |
| 查词回落 AI（本地词库未命中才扣） | 0.1/次 | 本地命中免费 |

> 注：全新视频字幕"一口价"已**打包**抓取+翻译，不再对该视频单独按句重复收翻译费。
> 具体数值在实现前需用真实 Supadata / DashScope / TTS 单价 × 目标毛利反推标定，本表为相对关系草案。

### 3.2 完全免费（不耗配额）

- 看**已缓存字幕**的视频 + **重看**（缓存命中，边际成本≈0）
- 本地 Whisper 转写兜底（自有算力）
- 本地词库命中的查词
- SRS / FSRS 卡片复习
- 收藏单词 / 视频（免费层限 **50** 个）
- 课程、发音页（phonics）、基础导航、阅读已有文章

> **已否决的旧思路**（记录备查）：早期讨论过"看新视频扣分""A2+ 阅读扣分"，后改为按 AI 计算量收费——更贴成本、更养免费盘、用户更好懂。

### 3.3 视频字幕收费：按"入口 × 缓存"判定

收费区分**入口**(谁承担抓取成本)和**缓存**(是否已生成):

| 入口 | 缓存命中 | 缓存未命中 |
|---|---|---|
| **Chrome 插件**（客户端直接抓 YouTube 字幕，对我们零成本，如 DejaVocab） | 免费 | **免费**（客户端抓，不走 Supadata） |
| **网站内**（iframe 跨域 → 服务端 Supadata 抓取 + 翻译，对我们有成本） | 免费 | **扣配额**（见 §3.1 一口价；扣触发者） |

- 复用 SUBS-003 的字幕 30 天 Redis 缓存判定命中。
- 热门视频在网站内抓一次 → 全平台 30 天免费;冷门按需付费,激励对齐,且防刷爆 Supadata 预算。
- **插件入口免费**是有意为之:鼓励用插件、降我们的成本、对标 DejaVocab 的客户端模式。
- **依赖**:Chrome 插件需具备客户端抓取 YouTube 字幕的能力（EXT-002 已做字幕提取）；网站内继续走 Supadata。
- **⚠️ 待确认（§9）**：中文翻译走 DashScope 是真成本，与入口无关。插件入口若也显示中文翻译，该翻译成本是"作为引流亏本承担"还是"仍按配额计量"，需 PM 拍板。

## 4. 功能权限门槛（配额之外的等级区分）

**原则：只列 Esponal 真实拥有的功能；未做的功能不作为当前卖点（见 §4.2 路线图）。**

### 4.1 当前真实的等级门槛

| 能力（均为现有功能） | 基础 | 进阶 | 高阶 / 终身 |
|---|---|---|---|
| 平台/本地字幕、基础查词、SRS-FSRS、看缓存视频 | ✅ | ✅ | ✅ |
| 收藏单词/视频上限 | 限 50 | 无限 | 无限 |
| **短语/搭配高亮 + 相关短语用法**（PHRASE-001 + LEX-003） | — | ✅ | ✅ |
| 新功能优先体验 / 优先技术支持（政策，非功能） | — | — | ✅ |
| 专属会员编号 / 共建者社区 | — | — | 仅终身 |

> **AI 加工类能力对所有等级开放、按配额计量**（高质量字幕抓取+翻译、AI 对话 talk、TTS、查词回落）——免费用户用注册赠送的 50 配额试用，用完需订阅。等级差异主要体现在**月度配额量（50 / 500 / 1000）** + 上面的硬门槛。
>
> **进阶 vs 高阶 的真实差异**：配额翻倍（500→1000）+ 新功能优先 + 优先支持。诚实地说，高阶相对进阶的差异目前偏薄，主要靠配额量；待 §4.2 路线图功能落地后会更厚。

### 4.2 已立项的高阶/终身专属功能（用户已确认要做，各自单独 ticket）

- **语感网络可视化**（`LEX-006`）——把相关词/搭配/同根词形连成网络图。底层数据已有（短语高亮 PHRASE-001 + 相关短语 LEX-003），本项做可视化。契合"靠语感/搭配习得"理念。
- **Anki 导出**（`VOCAB-013`）——生词本导出 Anki 牌组。**调性提醒**：VISION 反 SRS 刷卡、UI 禁用 Anki 字眼；定位为"导出到你习惯的外部工具"的便利功能（不改变站内"阅读输入、不打卡"主张）。

> **它俩落地后，作为高阶/终身专属门槛**——届时进阶 vs 高阶 的差异化从"主要靠配额量"变得扎实（高阶独占语感网络 + Anki 导出）。在它们上线前，定价页**不提前作为卖点**承诺。

## 5. 关键交互

- **扣费不弹窗**：余额充足时静默扣除，不打断。
- **积分不足**：**不弹模态框**；在触发处（视频卡片 / 功能入口）显示**内联提示**"配额不足 · 升级解锁" + 升级入口。
- **缓存命中免费**：用户感知不到扣费，体验顺滑。
- **配额可见**：账户页/顶栏显示当前配额余额 + 等级 + 本月刷新时间。

---

## 6. 数据模型要点

- **User 扩展**：`plan`（free / premium_m / premium_y / ultra_m / ultra_y / lifetime_premium / lifetime_ultra）、`planExpiry`、`creditBalance`（整数·分配额）、`lastRefillAt`、`builderNo`（终身专属编号）。
- **配额流水（审计/对账）**：`creditTransactions`（userId、delta、reason、refType[video/talk/tts/lookup/phrase/refill/grant]、refId、createdAt）。
- **月度刷新任务**：定时 job 按 plan 类型执行——订阅=覆盖为额度；终身=累加额度；免费=不动。
- **缓存判定**：复用现有字幕 Redis 缓存（SUBS-003），无需新建。
- **限量名额**：终身两档各自 `soldCount` / `limit=500`，用于进度条与售罄控制。

---

## 7. 不在本 spec 范围（各自单独排）

- ❌ **支付渠道集成**（微信支付 / 支付宝 / 订阅扣款）——独立大模块，**单独 spec**。本 spec 只做"积分引擎 + 等级 + 扣费 + 权限门槛 + 刷新"。
- ❌ **Anki 导出功能** 与 **语感网络可视化**——均为未做的路线图功能（§4.2），各自单独 ticket，且 Anki 需先过 PM 调性决策。
- ❌ **Chrome 插件客户端抓字幕降 Supadata 成本**——backlog 调研，单独评估。
- ❌ **生成全新分级读物**——功能未做，暂不纳入计费。

---

## 8. 验收/测试要点（供后续实现计划）

- 各动作正确扣对应配额（含小数精度·分配额存储）。
- 缓存命中=免费、未命中=扣费并生成。
- 余额不足时拦截动作 + 内联提示，不扣成负数。
- 月度刷新：订阅覆盖 / 终身累加 / 免费不动，三种正确。
- 升级即时生效；降级被拒；停订到期掉回免费层且订阅配额清零。
- 功能门槛按等级正确放行/拦截。
- 终身名额售罄保护。
