# WEB-010 — 登录后首页"继续学习"卡片

**优先级**：P1 | **负责人**：Codex1 | **日期**：2026-05-16
**前置依赖**：WEB-009（要复用 brand token + Hero 旁的位置）

---

## 背景

Claude2 评审：
> 用户登录后想接着昨天的单元/视频，没有任何位置告诉他从哪儿继续。"继续学习"是低成本高价值的功能，登录后首页顶部加一个"上次看到 unidad-3 / video XXX 03:42"的卡片，用现有数据就能做。

设计文档明确产品定位为"兴趣驱动学习伴侣，非强制打卡课"，所以这个卡片**不是任务待办列表，是温和的回到现场**。

---

## 一、数据来源

复用现有数据，不加新表：

### A. 上次看的视频
- 数据源：`WordEncounter` 表中 `userId` 最近一条 `sourceType = "video"` 的记录
- 拿 `sourceUrl`、`timestampSec`、`originalSentence`、`createdAt`
- 用 oEmbed 或视频卡片缓存补出 `videoTitle` + `thumbnail`

### B. 上次看的课程单元
- 数据源：`WordEncounter` 中最近一条 `sourceType = "course"` 的记录
- 拿 `courseRef`（格式 `unidad-N / xxx`）解析出 `slug = "unidad-N"`
- 从 `content/curriculum/unidad-N.json` 拿单元标题

如果用户既没看过视频也没进过课程 → 整个卡片不渲染（保持首页清爽）。
如果只有其一 → 单卡，不强行凑两张。

---

## 二、UI 规格

位置：登录后首页 Hero 区域的位置（HomeHero 不渲染时改渲染本组件），SiteHeader 下方、搜索栏上方。

布局（最多 2 张卡，并排）：

```
┌─ 继续学习 ────────────────────────────────────────────────┐
│                                                          │
│  ┌──────────────────────────┐  ┌──────────────────────┐  │
│  │ [缩略图 16:9]             │  │ unidad-3 标题        │  │
│  │ 视频标题（最多两行）       │  │ 上次：5 分钟前       │  │
│  │ 上次看到 02:34 · 5分钟前  │  │ [→ 继续单元]         │  │
│  │ [→ 继续观看]             │  └──────────────────────┘  │
│  └──────────────────────────┘                            │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

样式：
- 整体卡：`rounded-hero shadow-card bg-surface px-8 py-6`
- 标题"继续学习"：`text-sm font-semibold text-gray-500 uppercase tracking-wider`，左上对齐
- 卡内"视频卡"：`rounded-card border border-gray-100 hover:border-brand-300 transition`，缩略图 + 标题 + meta + 主按钮
- 卡内"课程卡"：emoji 📖 + 单元标题 + meta + 按钮，无缩略图（更轻）
- 主按钮：`bg-brand-500 hover:bg-brand-600 text-white rounded-card text-sm px-4 py-2`
- 时间戳格式："5 分钟前 / 2 小时前 / 昨天 / 3 天前"（用现有的 `formatRecentTime` 或写一个简版）

跳转链接：
- 视频：`/watch?v=<videoId>&t=<timestampSec>`
- 课程：`/learn/<slug>`

## 验收标准（Codex2 用）

1. 未登录访问 `/` → 显示 HomeHero（WEB-009），不显示 ContinueLearning
2. 登录后无任何 WordEncounter → `/` 显示空 hero 区或简单欢迎语，不显示 ContinueLearning
3. 登录且有视频 encounter → 显示视频卡，"继续观看"链接含 `&t=` 跳到正确时间
4. 登录且有课程 encounter → 显示课程卡，"继续单元"链接到 `/learn/unidad-N`
5. 两种 encounter 都有 → 并排两张卡
6. `npm test` 通过
7. `npm run build` 通过

---

## 注意事项

- 这个组件每次进首页都会查 DB，要确保 `WordEncounter` 表上有 `(userId, createdAt desc)` 的复合索引；如果没有，加 migration
- 缩略图用 `https://img.youtube.com/vi/<videoId>/mqdefault.jpg`，不需要再调 YouTube API
- 时间戳"5 分钟前"用纯函数 `formatRelativeTime(date)`，写在 `src/lib/dates.ts`，方便复用
- 不要在登录态下让卡片"消失"——如果数据查询失败，渲染一个 fallback 的"开始学习"提示卡，链接到 `/learn`
