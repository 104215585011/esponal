# CREDITS-FE-003 — 积分账户页:配额消耗说明区

**优先级**:P1(消耗透明度)
**实现**:Codex1(前端)· **测试**:Codex2 + 真机 · **验收**:Gemini1 视觉 + PM
**依赖**:`ACTION_COST_MINOR`(config,单一真源);积分账户页 `/account/credits`(CREDITS-FE-002 已建)
**设计模型**:`docs/tickets/CREDITS-cost-badge-mockup.html`
**预估**:0.5 天

## 背景
用户要求在**积分账户页里写清楚每个动作消耗多少配额**。在 `/account/credits` 页(FE-002)汇总卡下方、流水列表上方,新增一个"配额消耗说明"区。

## 要做
### A. 消耗说明区(放在汇总卡之后、流水之前)
逐条列出消耗动作 + 单价,**数值从 `ACTION_COST_MINOR` 取(÷100),不写死**:
| 动作 | 说明 | 消耗 |
|---|---|---|
| AI 对话 | 每轮对话 | 0.5 |
| 发音朗读 (TTS) | 每次朗读 | 0.1 |
| 查词(AI 回落) | 仅词典查不到、AI 补充释义时 | 0.1 |
| 短语提取 | 每句 | 0.05 |
| 视频字幕解锁 | 仅全新(未缓存)视频,不分时长 | 0.5 |

### B. 免费动作区
轻样式列出不消耗的:看缓存视频、本地词典查词、复习/SRS、收藏(限50)、重看已解锁字幕。

### C. 底注
"配额仅用于 AI 加工;费率以实际扣费为准,数值随版本可能调整。"

## 验收标准

- [ ] 单价**从 ACTION_COST_MINOR 读取并 ÷100 展示**(改 config 后页面自动同步;测试覆盖)。
