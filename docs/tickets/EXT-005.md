## 二、.crx 文件托管

### 当前限制
本仓库里 `extension/` 是源代码，没有打包好的 `.crx`。

### 本 ticket 做的
1. 在 `extension/` 加一个 `npm run package` 脚本，把 `dist/` 打成 `esponal-extension.crx`（或者退化成 `.zip`，更稳）
2. 把生成产物放到 `public/extension/esponal-extension.crx`（或 `.zip`）
3. 下载 CTA 直接 `<a href="/extension/esponal-extension.crx" download>`

如果时间紧 / 打包复杂：
- v1 直接放 `.zip`（用户自己解压拖入），文案改"下载安装包"
- v2 后续再加 `.crx` 自动签名

### 注意
`.crx` 需要私钥签名。如果用 Chrome 自带打包，私钥（`.pem`）**不能进 git**，应放 `.gitignore`，加到 `.env.example` 注释里说明本地手动打包流程。
