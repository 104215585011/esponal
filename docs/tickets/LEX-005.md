# LEX-005 — 修复现有动词的假变位（规则生成器留下的旧账）

**类型**：后端 / 数据修复
**实现**：Codex1
**测试**：Codex2 + PM 抽检
**前置依赖**：无（独立，但与 LEX-002 step 4 共用「真实变位生成」逻辑，建议同期做）
**预估**：0.5-1 天
**优先级**：中高（直接影响查词命中——不规则动词的变位查不到原型）

---

## 背景（PM 2026-05-29 在 LEX-002 调研时发现）

LEX-001 seed 阶段，动词的 `morphology` + `forms[]` 是用 `src/lib/conjugate.ts` 的 `tryConjugateVerb()` 生成的。该函数是**纯规则拼形**（`buildTense(lemma, endings)`：取词干 + 规则词尾），**完全不处理西语不规则变位**。

后果：所有不规则动词的变位都是**假形**。实证：

| 动词 | DB 现存（假）| 正确形 |
|---|---|---|
| poder | podo / podes / pode / podí / podió / poderé | **puedo / puedes / pude / pudo / podré** |
| querer | quero / queres / querí / quereré | **quiero / quieres / quise / querré** |
| estar | esto / estas / estó / estaron / estaste | **estoy / está / estuve / estuvieron / estuviste** |

**直接危害**：查词靠 `forms: { has: 'puedo' }` 命中原型。`poder` 的 forms 里只有假的 `podo`，所以学习者查 `puedo` **命中不了 `poder`**，回落 DashScope 实时生成（慢、不可控）。规则动词（hablar/comer 等）不受影响。

---

## 范围

- 现有 `kind=word` 且为动词的条目：**94 条**（`partOfSpeech` ∈ {verb, v.}）。
- 只修动词变位；名词/形容词的 morphology 不在本 ticket。
- construction 里的动词（gustar 等）也要顺带核查（gustar 变位虽特殊但常用形要对）。

---

## 实施

1. **改造或弃用 `tryConjugateVerb`**：纯规则拼形无法覆盖西语不规则，方案二选一（与 LEX-002 step 4 统一）：
   - **DeepSeek 生成真实变位**（与 LEX-002 step 4 同一套 prompt/校验，复用最省事）；或
   - 接成熟西语变位库（如有 license-clean 的，需 PM 核实许可证——同 LEX-002 的商用约束）。
2. 写脚本 `scripts/lexicon/refresh-verb-morphology.mjs --dry-run/--write`：
   - 拉出 94 个动词，逐个重新生成 `morphology` + 重建 `forms[]`
   - 默认 dry-run 打印 before/after 对比；`--write` 才落库
   - 包事务、`--resume` 断点续跑
3. **真实性抽检**（关键，别再交一批假形）：脚本对随机若干已知不规则动词，断言关键人称形正确（puedo/quiero/estoy/tengo/voy/soy/hago…），不通过则报错中止。
