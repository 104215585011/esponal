# WEB-008 — Transcript 虚拟化滚动 + 用户脱钩浏览

**优先级**：P1 | **负责人**：Codex1 | **日期**：2026-05-16
**前置依赖**：WEB-007 ✅

---

## 背景

当前 `TranscriptPanel` 在 commit `b96e44a` 后改成"全量渲染所有字幕 cue"。对于一条 30 分钟的视频（200~600 条 cue），首次进入页面会渲染上千个 React 子节点（每个 cue 还包含逐词 span + 高亮查询），首屏明显卡顿，且 `IntersectionObserver` / `scrollIntoView` 在长列表里也变慢。

但又不能回到旧的 ±4 cue 窗口——窗口太小用户根本无法手动往前/往后浏览未来或过去的字幕。

本 ticket 引入"虚拟化 + 用户脱钩浏览"模式，让首屏轻量、用户可自由翻看、播放跟随仍然平滑。

---

## 核心交互模型

**两种模式**：
1. **跟随模式（默认）**：active cue 推进时，TranscriptPanel 自动滚动让 active cue 居中。
2. **浏览模式（用户脱钩）**：用户用鼠标/触屏滚动后，TranscriptPanel 不再跟随播放进度；播放器**继续播放**，进度条/视频不受影响；右下角浮出「↺ 回到当前位置」按钮，点击恢复跟随模式。

**点击 cue 行**：在任一模式下都触发 `seekTo(cue.start)` 真实跳转播放。

**关闭 LookupCard**：在浏览模式下点击别处只关闭卡片，不跳转、不切回跟随。

---

## 渲染策略

### 渲染窗口（`renderRange`）

维护一个 `[renderStart, renderEnd]` 区间，只渲染这区间内的 cue。

**初始值**：
- `renderStart = 0`
- `renderEnd = min(subtitleCues.length, INITIAL_RENDER_COUNT)`，`INITIAL_RENDER_COUNT = 30`

**动态扩展**（向下加载更多 cue）：
- 在渲染列表末尾放一个底部哨兵 `<div ref={bottomSentinelRef} />`
- 用 `IntersectionObserver` 监听这个哨兵
- 当哨兵进入视口（用户滚动到接近底部）→ `renderEnd = min(length, renderEnd + LOAD_MORE_BATCH)`，`LOAD_MORE_BATCH = 30`

**向上加载**（用户向前回看）：
- 同样在列表顶部放一个顶部哨兵
- 进入视口时 `renderStart = max(0, renderStart - LOAD_MORE_BATCH)`
- **注意**：向上扩展会改变列表高度，需要保持当前用户视觉位置不跳——记录 `scrollTop` 和扩展前的 `scrollHeight`，扩展后用 `scrollTop += newScrollHeight - oldScrollHeight` 修正。

**播放跟随时强制扩展**：
- 当 `activeCueIndex` 接近 `renderEnd`（差距 < 5）时，自动 `renderEnd += LOAD_MORE_BATCH`
- 当 `activeCueIndex < renderStart`（用户跳到了已卸载的过去 cue）时，把 `renderStart` 缩到 `max(0, activeCueIndex - 5)`

## 跟随模式 vs 浏览模式

### state

```ts
const [followMode, setFollowMode] = useState(true);
```

### 切换规则

**进入浏览模式（followMode = false）**：
- 用户触发的 `wheel` / `touchmove` / `keydown(PageUp/PageDown/ArrowUp/ArrowDown)` 事件
- **不要用 `onScroll` 判断用户行为**——`scrollIntoView` 也会触发 `onScroll`，会误判
- 用 `pointerdown` + `wheel` + `touchstart` 事件判断"是用户主动操作"

**回到跟随模式（followMode = true）**：
- 点击「↺ 回到当前位置」按钮
- 切换字幕模式 tab（ES+中 / 仅西语 / 仅中文）时也重置为跟随
- 切视频时重置为跟随

### 跟随时的滚动

- 当 `followMode === true` 且 `activeCueIndex` 改变时：
  - 标记 `isProgrammaticScrollRef.current = true`
  - `cueRefs.current[activeCueIndex]?.scrollIntoView({ behavior: "smooth", block: "center" })`
  - 250ms 后清除标记

### 浏览时的播放

- 浏览模式下，**`activeCueIndex` 状态继续更新**（来自 `getCurrentTime` 轮询），但**不触发 scrollIntoView**
- 视频继续播放，进度条/iframe 完全不受影响

## 性能要求

- **首次进入页面**：仅渲染 ≤30 条 cue，首屏交互可用 < 200ms（凭感觉，不要求精确指标）
- **完整字幕长度**（500+ cue）页面也不卡
- 用户滚动 fps 接近 60
- LookupCard 浮层位置仍用 fixed + 锚点坐标（保持 commit `8c8f517` 的方案）

## 验收标准（Codex2 用）

1. **首屏轻量**：用临时 dev server 打开 `/watch?v=...`，DOM 中 `[data-cue-index]` 节点数 ≤ 35（说明虚拟化生效）
2. **滚动加载**：用 Playwright/CDP 模拟 wheel 滚动到底，DOM 节点数随滚动增加
3. **播放脱钩**：滚动后视频继续播放，`getCurrentTime()` 正常增长，进度条不被拉动
4. **回到当前**：点击浮动按钮后，列表滚回 active cue 居中位置，按钮消失
5. **点击跳转**：点击任一 cue 行触发 `player.seekTo(start)`，视频跳转
6. **`npm test`** 通过，`npm run build` 通过
