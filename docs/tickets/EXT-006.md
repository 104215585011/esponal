## 背景

YouTube 2024-2025 反爬升级后：
- Vercel / 任何 datacenter IP 调 YouTube player API 一律被拦
- yt-dlp 本地无 cookie 也被拦，cookie 导出对当前环境不可行
- Apify 还能用但持续付费 + 中间字幕仍有洞

唯一**不被反爬豁免**的请求来源是「真实用户的真实浏览器」。我们已经有 Chrome 插件——本票把它扩展成**字幕收割器**：用户打开任何 YouTube 视频页，插件用用户的 session fetch 字幕，POST 到我们后端 Redis。第一个安装插件的人访问视频 = 给所有后续访问者预热缓存。

EXT-002 现有的 DOM `MutationObserver` 字幕监听**不复用**——那条路只能在播放时一行行抓，无法整段 harvest。本票走 `ytInitialPlayerResponse.captions.playerCaptionsTracklistRenderer.captionTracks` 路径，1 秒拿全片。

## 一、Extension 端

### 新建 `extension/harvest.js`

content script，匹配 `https://www.youtube.com/watch?v=*`，在页面加载完成后执行：

```js
// 伪代码
(function harvestSubtitles() {
  const videoId = new URLSearchParams(location.search).get("v");
  if (!videoId) return;

  // 防重复：同一 tab 一个 videoId 只 harvest 一次
  if (window.__esponalHarvested === videoId) return;
  window.__esponalHarvested = videoId;

  // ytInitialPlayerResponse 是 YouTube 在 HTML 里注入的全局变量
  // 我们读不到 page-script 的 window，需要注入一个 page script 把它桥接出来
  const script = document.createElement("script");
  script.textContent = `
    window.postMessage({
      type: "esponal-player-response",
      data: window.ytInitialPlayerResponse?.captions
        ?.playerCaptionsTracklistRenderer?.captionTracks ?? []
    }, "*");
  `;
  (document.head || document.documentElement).appendChild(script);
  script.remove();

  window.addEventListener("message", async (event) => {
    if (event.data?.type !== "esponal-player-response") return;
    const tracks = event.data.data;

    // 只挑 es / es-* 的轨道
    const esTracks = tracks.filter((t) => t.languageCode?.startsWith("es"));
    if (esTracks.length === 0) return;

    for (const track of esTracks) {
      try {
        const url = track.baseUrl + "&fmt=json3";
        const res = await fetch(url, { credentials: "include" }); // 用户 cookie 自动带上
        if (!res.ok) continue;
        const json = await res.json();
        const cues = parseJson3ToCues(json);
        if (cues.length < 5) continue; // 太少不算数据

        await fetch("https://<your-domain>/api/subtitle/ingest", {
          method: "POST",
          headers: {
            "Content-Type": "application/json",
            "X-Esponal-Ingest-Token": "<EXT_INGEST_TOKEN from manifest>"
          },
          body: JSON.stringify({
            videoId,
            lang: track.languageCode.startsWith("es-") ? track.languageCode : "es",
            cues
          })
        });
      } catch (err) {
        console.warn("[esponal harvest]", err);
      }
    }
  }, { once: false });
})();
```

### `parseJson3ToCues`

YouTube json3 字幕格式：
```json
{ "events": [
  { "tStartMs": 1240, "dDurationMs": 2800, "segs": [{ "utf8": "Hola a todos" }] },
  ...
]}
```

转成我们的 `SubtitleCue`：
```ts
{ start: tStartMs / 1000, dur: dDurationMs / 1000, text: segs.map(s => s.utf8).join("") }
```

过滤：跳过空 segs、跳过 dur <= 0 的 cue。

### `extension/manifest.json` 改动

```json
{
  "content_scripts": [
    {
      "matches": ["https://www.youtube.com/*"],
      "js": ["content.js"]
    },
    {
      "matches": ["https://www.youtube.com/watch*"],
      "js": ["harvest.js"],
      "run_at": "document_idle"
    }
  ],
  "permissions": ["storage"]
}
```

`EXT_INGEST_TOKEN` 在打包时通过 `process.env.EXT_INGEST_TOKEN` 注入到 harvest.js（参考 esbuild define 配置）。

### `extension/popup.html` / `popup.js`

加一行状态提示：「最近一次收割：YYYY-MM-DD HH:MM, videoId=xxx」。从 `chrome.storage.local` 读最后一次 harvest 元信息。这是用户向反馈，让用户知道插件在干活。

## 二、后端 ingest 路由

### 新建 `src/app/api/subtitle/ingest/route.ts`

```ts
import { NextResponse } from "next/server";
import { redis } from "@/lib/redis";
import { checkRateLimit, getRetryAfterSec, ingestLimiter } from "@/lib/ratelimit";

const SUBTITLE_CACHE_TTL = 86400 * 30; // 30 天
const MAX_CUES = 5000;
const MIN_CUES = 5;
const MAX_PAYLOAD_BYTES = 500_000;

type IngestBody = {
  videoId: string;
  lang: string;
  cues: Array<{ start: number; dur: number; text: string }>;
};

export async function POST(request: Request) {
  // 1. 共享 token 鉴权
  const token = request.headers.get("x-esponal-ingest-token");
  if (!token || token !== process.env.EXT_INGEST_TOKEN) {
    return NextResponse.json({ error: "unauthorized" }, { status: 401 });
  }

  // 2. 限流（防即使 token 泄露也无法 DDoS）
  const rl = await checkRateLimit(ingestLimiter, request, null);
  if (!rl.allowed) {
    return NextResponse.json(
      { error: "rate limited", retryAfterSec: getRetryAfterSec(rl.reset) },
      { status: 429 }
    );
  }

  // 3. 体积上限
  const raw = await request.text();
  if (raw.length > MAX_PAYLOAD_BYTES) {
    return NextResponse.json({ error: "payload too large" }, { status: 413 });
  }

  const body = JSON.parse(raw) as IngestBody;

  // 4. 格式校验
  if (
    typeof body.videoId !== "string" ||
    body.videoId.length > 32 ||
    !/^[\w-]+$/.test(body.videoId) ||
    typeof body.lang !== "string" ||
    body.lang.length > 12 ||
    !Array.isArray(body.cues) ||
    body.cues.length < MIN_CUES ||
    body.cues.length > MAX_CUES
  ) {
    return NextResponse.json({ error: "invalid body" }, { status: 400 });
  }

  // 5. 内容净化：每条 cue
  const cleanCues = body.cues
    .filter(
      (c) =>
        typeof c.start === "number" &&
        typeof c.dur === "number" &&
        typeof c.text === "string" &&
        c.start >= 0 &&
        c.dur > 0 &&
        c.dur < 600 &&
        c.text.length > 0 &&
        c.text.length < 500
    )
    .map((c) => ({ start: c.start, dur: c.dur, text: c.text.trim() }));

  if (cleanCues.length < MIN_CUES) {
    return NextResponse.json({ error: "not enough valid cues" }, { status: 400 });
  }

  // 6. Write-once 策略：只有 key 不存在时才写
  // 防止恶意覆盖。如果有真实质量问题，由 PM 手动清缓存重收。
  const cacheKey = `subtitle:${body.videoId}:${body.lang}`;
  const existing = await redis.get(cacheKey);
  if (existing) {
    return NextResponse.json({ ok: true, status: "exists" });
  }

  await redis.set(cacheKey, JSON.stringify(cleanCues), "EX", SUBTITLE_CACHE_TTL);

  return NextResponse.json({
    ok: true,
    status: "ingested",
    cueCount: cleanCues.length
  });
}
```

## 四、安全模型

| 威胁 | 缓解 |
|---|---|
| 任意人发 POST 污染缓存 | `X-Esponal-Ingest-Token` 共享密钥（写在 extension 打包产物里，公开但每月可轮换） |
| Token 泄露后污染 | Write-once 写入（已有 cache 不覆盖）+ cue 格式 / 大小校验 |
| 单 IP 海量灌 | rate limit 30 req/min per IP |
| 错误数据 stuck | TTL 30 天自动过期；管理脚本可手动 `DEL` 某 key 强制重收 |
| Harvest 影响用户隐私 | 不上报 userId / cookie / videoTitle，仅 `{videoId, lang, cues}` |

**坦诚说**：token 在公开 .crx 里可被反编译拿到。这不算"安全"，只是"摩擦层"。真攻击者还是能 POST。最终保护机制是 write-once + 格式校验 + TTL，让最坏情况下的损害有界。对于个人项目这个层级够了。

## 注意事项

- harvest.js 不能直接读 `window.ytInitialPlayerResponse`（content script 在 isolated world）。必须 inject 一段 page script 通过 `postMessage` 桥接。已在伪代码示例
- json3 字幕里可能含 HTML 实体（`&amp;`、`&#39;`）—— `parseJson3ToCues` 解码这些
- 某些 YouTube 视频字幕用 `srv3` 而非 `json3`，加 `fmt=json3` 强制 YouTube 转码
- `credentials: "include"` 必须有，否则浏览器不带 cookie，又会触发 bot 检测
- 不要 harvest 用户的隐私模式视频（虽然多半也不在我们策划频道里）
