# MOBILE-001 — watch 页 + 字幕面板 移动端独立布局重设计

**优先级**：P1（移动端重设计 epic 的第一张）
**区域**：watch / `src/app/watch/WatchClient.tsx` + `SubtitlePanel.tsx` + `TranscriptPanel.tsx` + `LookupCard.tsx`
**设计**：Gemini1（移动端设计稿 `docs/tickets/MOBILE-001-design.md`）
**实现**：Codex1
**测试**：Codex2 + Gemini1 评审 + PM 验收
**预估**：2-3 天（含设计）

---

## 背景与决策

- 产品最终目标：上架 Android / iOS app（Capacitor 路线，打包放最后）。当前阶段：**先把网页移动端做好做稳**，打包不在本轮。
- 桌面端（网页）已基本完成。**移动端 UI 需要重新设计**。
- **PM + 用户决策（2026-06-01）**：移动端走**独立布局 / 组件**（不是只加 CSS 断点适配），追求移动端原生级体验。
- 第一优先：**watch 页 + 字幕面板**（用户使用频率最高的核心页）。

## 现状（重设计的起点）

`WatchClient.tsx` 当前用 `lg:` 断点在同一组件里条件渲染：
- **桌面**：视频 + 字幕浮层(SubtitlePanel isOverlay) / 右侧 560px `TranscriptPanel` / 滑出 `WatchSidebar`。
- **移动**：视频 + 一个 tab 切换器（字幕 / 转写 / 推荐），内容堆在下面。
- 移动端是"塞进同一组件、CSS 隐藏桌面版"的**临时方案**，tab 体验粗糙——本票要把它**重新设计成独立、精心设计的移动端**。

## 目标用户与设计调性（设计前提，2026-06-01 PM 与用户确认）

- **主力用户：成人 / 学生自学者**（高效、能坚持、为体系和学习效果付费）。
- **明确不做儿童早教向**：续费/使用决策在家长手里、不可控，不作为主力。所以移动端**不要游戏化/卡通/大幼龄化**那一套。
- **调性**：成人高效向 —— 信息密度合理但干净、专业、克制；优化碎片时间学习、上手快、可坚持。
- 战略背景：产品学习阶梯已完整（发音/词汇/阅读/视频/课程/对话全都有），当前杠杆是**移动端体验 + 留存打磨**，不是加功能。打法是"成人西语学习这个窄市场里做体验第一"。

## ⚠️ 架构硬约束（实现必读）

1. **只有一个 YouTube player 实例**（`PLAYER_IFRAME_ID = "esponal-youtube-player"`）。字幕/转写/查词面板都只接收共享状态（`currentTimeSec` / `activeLookup` / `playbackRate` / `activeCue` 等）。
2. **移动端独立布局 ≠ 另起一页**：是**同一个 player + 同一套状态**的不同空间排布。**绝不能渲染出第二个 player**（重复 id / 双播放器 / 状态打架）。
3. **推荐做法**：把布局拆成 `WatchDesktopLayout` / `WatchMobileLayout` 两个展示组件，**共享状态和 player 逻辑仍留在 `WatchClient`（或提到自定义 hook）**，按视口选一个挂载。具体方案 Codex1 定，但必须满足约束 1、2。
4. 与 **WATCH-009**（字幕下载 PDF 按钮，放字幕面板右侧）协调：移动端字幕面板形态变了，下载按钮在移动端的落点要在本设计稿里一并定，避免两票打架。**建议 MOBILE-001 设计先行**，WATCH-009 的移动端按钮位置参照本设计稿。

## 待 Gemini1 设计的核心问题（写进 MOBILE-001-design.md）

- **整体移动端范式**：保留 tab？还是改可滚动堆叠 / 底部抽屉(bottom sheet) / 视频吸顶 + 下方内容流？给出明确选型与理由。
- **视频与面板的空间分配**：竖屏下视频多高、字幕/转写区多高、是否视频吸顶滚动。
- **字幕面板（SubtitlePanel）移动端形态**：当前显示模式切换、播放速度、逐词点击查词的触摸目标大小。
- **转写面板（TranscriptPanel）移动端形态**：虚拟化长列表在移动端的滚动、当前句高亮、点击跳转。
- **查词卡（LookupCard）移动端形态**：桌面是卡片，移动端建议底部抽屉？查词时视频暂停的交互。
- **控制条**：全屏按钮、刷新字幕、速度等在移动端怎么摆（拇指可达区）。
- **横屏 / 全屏**：移动端横屏看视频 + 字幕浮层的处理。
- 触摸目标 ≥ 44px、安全区(刘海/底部 home bar)留白等移动端规范。

## 用户可见行为（目标）

- 在手机上打开 `/watch?v=...`，得到为移动端专门设计的布局：视频、字幕、转写、查词都好用、好点、不拥挤。
- 所有现有功能在移动端可用：逐词查词、显示模式切换、速度、转写跳转、推荐、（WATCH-009 后）字幕下载。
- 桌面端体验**不回退**（独立组件，互不影响）。
