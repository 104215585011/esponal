# OPS-001 — 错误监控接入（Sentry）

**优先级**：P2 | **负责人**：Codex1 | **日期**：2026-05-16
**前置依赖**：无

---

## 背景

生产环境出错只能去 Vercel logs 一行行翻 `console.error`，没有：
- 趋势图（错误率随时间变化）
- 错误分组（同一个 stack 出现 N 次）
- 用户行为路径（出错前点击了什么）
- 受影响用户数

每次 hotfix 都靠用户截图反馈，PM 无法主动发现问题。

本 ticket 接入 Sentry（也可换 LogRocket / PostHog，但 Sentry 免费额度对单人项目足够：5K errors/month）。

---

## 一、接入范围

### Next.js 端
- 客户端错误（React render、未捕获 Promise rejection）
- 服务端错误（API route throw、SSR throw）
- 性能跟踪先关掉（免费额度紧张）

### Chrome 插件端
- 不接（独立分发、隐私敏感、用户基数小）

### 关键自定义事件
除自动捕获外，主动上报：
- `/api/translate` Tencent 调用失败（区分 degraded vs throw）
- `/api/vocab/lookup` GLM-5 调用失败
- `/api/subtitle` Apify 抓字幕失败
- 任何 429 rate limited（OPS-002 之后）

---

## 二、技术方案

### 依赖

```bash
npm install @sentry/nextjs
```

### 初始化

按官方 wizard 走：

```bash
npx @sentry/wizard@latest -i nextjs
```

它会生成：
- `sentry.client.config.ts`
- `sentry.server.config.ts`
- `sentry.edge.config.ts`
- 改 `next.config.mjs` 包一层 `withSentryConfig`

### 环境变量（`.env.example`）

```
SENTRY_DSN=https://...@o0.ingest.sentry.io/0
SENTRY_AUTH_TOKEN=...          # 仅 CI 用，本地不需要
SENTRY_ORG=esponal
SENTRY_PROJECT=esponal-web
NEXT_PUBLIC_SENTRY_DSN=$SENTRY_DSN  # 客户端用
```

### 自定义事件 helper

```typescript
// src/lib/monitor.ts
import * as Sentry from "@sentry/nextjs";

export function reportTranslateFailure(text: string, error: unknown) {
  Sentry.captureException(error, {
    tags: { feature: "translate" },
    extra: { textLength: text.length, textPreview: text.slice(0, 40) },
  });
}

export function reportLookupFailure(word: string, error: unknown) {
  Sentry.captureException(error, {
    tags: { feature: "lookup" },
    extra: { word },
  });
}

export function reportSubtitleFailure(videoId: string, error: unknown) {
  Sentry.captureException(error, {
    tags: { feature: "subtitle" },
    extra: { videoId },
  });
}
```

把现有的 `console.error("Subtitle translation failed", error)` 等改为同时调用 `reportXxxFailure(...)`。`console.error` 保留（Vercel logs 还能用）。

### 用户上下文

API route 拿到 session 后：
```typescript
Sentry.setUser({ id: session.user.id });
```

匿名用户不上 user context（避免给一堆 anonymous 卡 quota）。

## 三、隐私

- 不要把字幕原文、用户输入的密码、生词原句作为 `extra` 全文上报
- `textPreview` 截前 40 字符够 debug
- 不要捕获 React props（默认行为可能泄露 token）

## 六、验收（Codex2 用）

1. `npm test` 通过
2. `npm run build` 通过（注意 Sentry build-time 上传 source map 可能失败，需要 `SENTRY_AUTH_TOKEN`；缺失时应只 warning 不报错）
3. 本地 dev `throw new Error("test sentry")`，Sentry dashboard 收到事件
4. 隐藏字幕原文：触发翻译失败后，Sentry 事件里 `extra` 只有 `textPreview` ≤ 40 字符
