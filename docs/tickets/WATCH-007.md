## 背景 / 需求

用户希望在右侧字幕工具条（现有 `ES+中 / 仅西语 / 仅中文 / 点击字幕跳转`）增加两项能力：

1. **加载方式切换**：现在字幕只有 WATCH-004 的「句子级合并」一种渲染。用户想保留**之前那种「逐 cue 逐行」的加载方式**作为可选项，让用户在两种之间切换。
2. **字幕下载按钮**：把当前字幕下载下来（PDF，带时间戳）。

---

## 用户可见行为

- 工具条出现「加载方式」切换（句子级 / 逐行）。
  - **句子级**（默认）：WATCH-004 行为，相邻 cue 合并成句、中文按句显示。
  - **逐行 / 逐 cue**：恢复改造前的逐 cue 渲染，每个 cue 一行、中文逐行。
  - 两种模式下**逐词查词、当前行高亮、自动滚动跟随都保留**。
  - 用户选择应**记忆**（localStorage），下次打开沿用。
- 工具条出现「下载」按钮：
  - 导出格式 **PDF**。
  - 内容**跟随当前显示模式**：ES+中 → 双语；仅西语 → 只西语；仅中文 → 只中文。
  - 每条**带时间戳**。
  - 中文显示正常（不乱码、不缺字）。

## 实施要求

### 1. 加载方式切换（逻辑 + 展示）
- 新增状态 `transcriptMode: "sentence" | "cue"`，默认 `"sentence"`，持久化到 localStorage。
- `"sentence"` 走现有 `groupCuesIntoSentences` 渲染路径（WATCH-004）。
- `"cue"` 走逐 cue 渲染：改造前的逐 cue 渲染逻辑在 git 历史里（WATCH-004 实现 commit `bf7acd6` 之前的 `TranscriptPanel.tsx`），可据此恢复一条逐 cue 渲染分支。
- 两路共用同一套查词 / 高亮 / 滚动逻辑，避免分叉成两套交互。

### 2. 字幕下载（PDF，带时间戳）
- 在工具条加「下载」按钮，点击导出 PDF。
- 内容随当前 `displayMode`（三选一）+ `transcriptMode`（句子级/逐cue 决定每条粒度）。
- 每条前缀时间戳（如 `0:42`）。
- **技术选型（PM 倾向，留设计/实现定夺）**：
  - ⚠️ 纯前端 `jsPDF` 直接生成 PDF 需嵌入 CJK 字体（几 MB），显著撑大 bundle，**不推荐**。
  - ✅ **推荐**：构建「打印友好视图」+ `window.print()`，用户在系统打印对话框「另存为 PDF」。天然支持中文、零字体负担、跟随显示模式只需一套打印样式。代价：走打印对话框而非一键直下。
  - 若坚持一键直下 PDF，需评估 CJK 字体子集化方案，单独评估成本。

### 3. 不回归
- WATCH-004 句子级渲染、三种显示模式、逐词查词、当前行高亮、自动滚动跟随全部保留。
- WATCH-005/006 的播放器布局与悬浮字幕改动不受影响。
