# WATCH-002 — 视频播放页重构

**类型**：UI 改造
**设计**：Gemini1（出 `docs/tickets/WATCH-002-design.md`）
**实现**：Codex1（按设计稿写代码）

**最终验收**：Claude1

**预估**：5-7 天
**优先级**：高（视频是用户输入西语的主要场景）

---

## 背景

当前视频播放页（`src/app/watch/*`）功能堆叠：

```
src/app/watch/
├── LookupCard.tsx
├── RelatedPanel.tsx
├── SubtitlePanel.tsx
├── TranscriptPanel.tsx
├── WatchSidebar.tsx
└── page.tsx
```

痛点：
1. **几个 Panel 之间没有清晰的视觉关系**——字幕、相关视频、转写、查词四个区域各做各的
2. **字幕样式没设计语言**——和站点其他部分割裂
3. **查词触发后视频是否继续播放，行为不一致**
4. **移动端体验不明**——需要确认视频 + 字幕 + 查词在小屏幕上的优先级

---

## 目标用户体验

实现后，用户应该感觉：

1. **视频是主角**——其他元素围绕它服务，不抢戏
2. **字幕清晰可读**——字号、对比度、位置都考虑过
3. **点词查询不破坏观看流**——视频默认暂停、卡片用完即走、播放无缝继续
4. **能快速跳到任意字幕时间点**（转写面板的核心价值）
5. **看完一个视频后，自然过渡到下一个**

## 具体要做的事

### 1. 整体布局重构
桌面端：
```
┌────────────────────────────────────────┐
│  视频（16:9 主区域）                    │
│                                        │
├────────────────────────────────────────┤
│  下方：字幕条 + 控制（语速/翻译切换）    │
└────────────────────────────────────────┘
┌─────────────┬──────────────────────────┐
│  转写面板    │  右侧栏（查词 Dock /     │
│ （可滚动）    │   相关视频 / 笔记切换）  │
│             │                          │
└─────────────┴──────────────────────────┘
```

移动端：
- 视频置顶
- 下方是 tab 切换：字幕 / 转写 / 查词 / 相关
- 不要把所有 Panel 同时塞进小屏幕

### 2. 字幕样式统一
- 字幕和站点字体一致（继承 NAV-001 / LECTURA-002 的字体语言）
- 双语字幕（中 / 西）排版清晰，中文不抢戏（小一号、灰色）
- 已收藏词在字幕里用低调虚线（同 LECTURA-002 的处理）
- 字幕字号可调

### 3. 查词流程优化
- 点击字幕里的词 → 视频自动暂停 → 右侧 Dock 显示查词卡（桌面）/ 弹出底部抽屉（移动）
- 用户关闭卡片 → 视频自动恢复播放
- 「+1 encounter」机制（VOCAB-012）自然继承，无需额外改

### 4. 转写面板交互
- 整段字幕按时间排列，当前播放段高亮
- 点击任意段 → 视频跳到该时间点
- 自动滚动跟随当前播放位置（用户手动滚动后停 5 秒自动恢复跟随）

### 5. 视频结束后的体验
- 视频自然结束 → 右下角显示"下一个推荐"卡片，用户点击进入
- 不要全屏覆盖（避免打断）
- 不要倒计时自动跳转（避免压力）

### 6. 速度控制
- 速度切换（0.75x / 1x / 1.25x）放显眼位置
- 西语学习者经常需要慢放
