# NAV-001 — 整站导航重构

**类型**：UI 改造
**所属**：Gemini1（UI 总监，前端实现）
**前置依赖**：无
**预估**：3-5 天
**优先级**：高（影响每个页面）

---

## 背景

当前导航（`src/app/components/web/SiteHeader.tsx`）的问题：

1. **移动端体验差**——只有顶部一条 sticky header，所有导航链接挤在一行；小屏幕下搜索框直接 `hidden`，没有 fallback
2. **缺少当前页面指示**——用户点进任意子页面，无法在 nav 里看出"我现在在哪"
3. **导航层级扁平**——发音 / 词汇 / 语法 / 阅读 / 视频 / 对话 / 拆解 / 词库 都是平行链接，没有视觉层级
4. **搜索 affordance 弱**——搜索框是 "搜索西语视频..."，但实际上搜索范围应该更广（视频 + 阅读 + 课程 + 词库）

---

## 目标用户体验

实现后，用户应该能：

1. **任何屏幕宽度**都能舒服地访问所有导航目标（桌面 + 移动端）
2. **一眼看出**自己当前在哪个板块
3. **快速跳转**到任何主要功能区，不需要先回首页
4. 搜索是**全局**的，不仅限于视频

---

## 强制约束

⚠️ **开工前必读：`docs/UI-DESIGN-CONSTRAINTS.md`**

- 不要给"今天还没学习"加角标
- 已登录用户头像旁不要放"等级"

## 具体要做的事

### 1. 桌面端 nav 重新设计
- 保留 logo + 主导航 + 搜索 + 用户菜单的基本结构
- **新增：当前页面 active 状态**（下划线 / 加粗 / 颜色变化任一种）
- 搜索框 placeholder 改为"搜索内容..."（更通用），点击聚焦后展开为更宽的输入框
- 用户菜单保留现状

### 2. 移动端 nav（核心改造）
- 顶部保留：logo + 搜索图标 + 汉堡菜单 + 用户头像
- 搜索图标点击 → 全屏搜索覆盖层（不是隐藏）
- 汉堡菜单点击 → 抽屉式侧边栏，列出所有主导航 + 当前页面指示
- 抽屉关闭：点击遮罩 / 点击导航项 / 滑动

### 3. 主导航的语义分组（不一定要 UI 上分组，但代码上要有结构）

把现在的扁平链接按语义重新组织：

```ts
// 概念上分三组
- 学习入口：发音 / 词汇 / 语法 / 阅读 / 视频 / 对话
- 工具：拆解器 / 词库
- 我的：（已登录可见）个人页 / 设置
```

UI 上要不要做视觉分组，由 Gemini1 决定，但代码结构要清晰。

### 4. 搜索范围扩展
- 当前 `action="/search"` 是搜视频
- 改造为全局搜索路由（如果后端 `/api/search` 不存在，先把 UI 做成"提示用户搜什么"的占位即可，**搜索 API 不在本 ticket 范围**）
