# WEB-013 — 移动端导航（汉堡菜单）

**优先级**：P1 | **负责人**：Codex1 | **日期**：2026-05-19
**状态**：`ready_for_dev`
**前置依赖**：WEB-009 ✅（brand token）

---

## 背景

PM 决定走「PWA + TWA」移动端路线（见 PWA-001）。但 PWA 化之前先填一个洞——**手机上现在根本没有导航菜单**。

`src/app/components/web/SiteNav.tsx` 当前写法：

```tsx
<nav className="hidden items-center gap-1 lg:flex">
```

`hidden lg:flex` = **< 1024px 直接不渲染**。手机用户进了首页之后只能靠手动改 URL 跳「课程 / 阅读 / 语法 / 词库」，等于 5 个一级入口全部消失。

---

## 一、设计规格

桌面（≥1024px）保持现状不动。手机 / 平板（<1024px）：

```
┌─────────────────────────────────────┐
│ [E] Esponal      🔍   [☰]    [头像] │  ← 顶栏：logo + 搜索图标 + 汉堡 + 头像
└─────────────────────────────────────┘

点☰ → 全屏覆盖式抽屉从右滑入：

┌─────────────────────────────────────┐
│                            [✕]      │
│                                     │
│   视频                               │
│   课程                               │
│   阅读                               │
│   语法                               │
│   词库                               │
│                                     │
│   ──────                            │
│   退出登录（如已登录）                 │
└─────────────────────────────────────┘
```

样式要求（沿用 WEB-009 brand token）：
- 抽屉容器：`fixed inset-0 z-50 bg-surface`
- 关闭按钮 ✕：右上角，`text-gray-500 hover:text-gray-900`
- 菜单项：`text-xl font-medium text-gray-800` + active 时 `text-brand-600`，纵向 stack、`py-4 px-6`、点击区域大
- 抽屉打开时 body 锁定滚动（`overflow-hidden`）
- 进入/退出动画：`transition-transform duration-200 translate-x-full → translate-x-0`
- 点抽屉外区域 / 按 ESC → 关闭

桌面端搜索框在 `< lg` 时**收成图标**（点击图标展开搜索 input），避免顶栏挤爆。如果实现复杂可以 v1 先让搜索框完全隐藏在抽屉里。

## 注意

- 必须 `"use client"`（要 useState）
- 用 `document.addEventListener("keydown", ...)` 而不是 React onKeyDown（按钮可能没 focus）
- 锁滚动用 `useEffect` 操作 `document.body.style.overflow`，组件卸载时复原
- 抽屉内的 `<Link>` 点击后**手动**调 setOpen(false)，否则 SPA 路由切换抽屉不会自己关
