# WEB-014 — 详情页返回按钮

**优先级**：P1（PWA standalone 模式下用户被困）
**区域**：web / navigation
**依赖**：WEB-009 ✅（设计令牌）、WEB-013 ✅（移动导航）

---

## 背景

所有详情页（Lectura 故事、课程单元、视频播放、SRS 复习、语法详情）**没有任何"返回上级"的入口**。普通浏览器还能用后退键凑合，但：

- PWA 装到桌面/手机后是 standalone 模式，**浏览器后退键不存在**
- 手机 Safari/Chrome 全屏滑动手势在某些设备/姿势下失效
- 用户从外部链接首次进入详情页时，浏览历史里没有上级

用户实测反馈："点进去出不来。"

---

## 用户故事

- 用户从 `/lectura` 列表点开一个故事 → 故事详情页顶部显示 `← 阅读`
- 点击 `← 阅读` 回到 `/lectura` 列表
- 课程单元、视频、复习、语法详情同样有对应返回按钮
- 行为是**固定回上级**（不依赖浏览历史），首次直接打开链接也能用

---

## 触动页面与目标路径表

| 当前路径 | 返回按钮文案 | 跳转目标 |
|---------|-----------|---------|
| `/lectura/[slug]` | `← 阅读` | `/lectura` |
| `/learn/[slug]` | `← 课程` | `/learn` |
| `/watch?v=...` | `← 首页` | `/` |
| `/vocab/review` | `← 词库` | `/vocab` |
| `/grammar/[topic]`（如存在） | `← 语法` | `/grammar` |
| `/extension`（如有详情子页） | `← 首页` | `/` |

**注**：`/vocab`、`/learn`、`/lectura`、`/grammar` 这些**列表页本身不加返回按钮**——它们是顶级导航目的地，SiteHeader 已经提供了所有顶级页面入口。

## 设计方案

### 共享组件：`src/app/components/web/BackLink.tsx`

新建一个客户端无关的纯展示组件（Next.js `Link` 即可，不需 `"use client"`）：

```tsx
import Link from "next/link";

type BackLinkProps = {
  href: string;
  label: string;
};

export function BackLink({ href, label }: BackLinkProps) {
  return (
    <Link
      href={href}
      className="inline-flex min-h-[44px] items-center gap-1.5 text-sm font-medium text-gray-500 transition-colors hover:text-gray-900 -ml-2 px-2"
      data-testid="back-link"
    >
      <span aria-hidden className="text-base leading-none">←</span>
      <span>{label}</span>
    </Link>
  );
}
```

**视觉规格**：
- 字号 `text-sm`（14px），灰色 `text-gray-500`，hover 加深 `text-gray-900`
- 箭头 `←` 字符 + 中文标签，gap 6px
- 触控区 `min-h-[44px]`，满足 iOS HIG 最小可点击区域
- `-ml-2 px-2` 让箭头视觉上与下方标题左对齐（padding 给触控区，负 margin 抵消视觉缩进）
- **不用 brand 色**——返回是次要动作，避免和"待复习"徽章等主操作抢视觉

### 放置位置

每个详情页在 `<SiteHeader />` 之下、`<header>`（页面标题区）之上插入：

```tsx
<main className="min-h-screen bg-app text-gray-900">
  <SiteHeader />
  <section className="mx-auto max-w-2xl px-4 py-10 sm:px-6">
    <BackLink href="/lectura" label="阅读" />
    <header className="mb-8 mt-4">
      <h1 className="text-2xl font-bold">{story.title}</h1>
      {/* ... */}
    </header>
    {/* page content */}
  </section>
</main>
```

每页一行，5 处分别 hardcode `href` + `label`，不做动态推断（KISS）。

### Mobile / Desktop 一致性

- 移动端：BackLink 占一整行，触控舒适
- 桌面端：同样位置，14px 文字不占视觉重量
- 不做特殊响应式分支

### 不做的

- 不实现"浏览器后退"行为（用户已确认要固定回上级）
- 不在列表页加返回按钮（顶级导航目的地）
- 不动 SiteHeader 自身
- 不做面包屑（breadcrumb），就一个返回链接

## 不在本票范围

- 浏览器后退键模拟（方案 Y/Z 都被排除）
- 面包屑导航
- "返回到具体进入路径"（如 watch 页根据 referer 决定回 / 或 /search）
- SiteHeader 自身改造
- 动画过渡
