# WEB-014 — 详情页返回按钮

**优先级**：P1（PWA standalone 模式下用户被困）
**区域**：web / navigation
**依赖**：WEB-009 ✅（设计令牌）、WEB-013 ✅（移动导航）

---

## 背景

所有详情页（Lectura 故事、课程单元、视频播放、SRS 复习、语法详情）**没有任何"返回上级"的入口**。普通浏览器还能用后退键凑合，但：

- PWA 装到桌面/手机后是 standalone 模式，**浏览器后退键不存在**
- 手机 Safari/Chrome 全屏滑动手势在某些设备/姿势下失效
- 用户从外部链接首次进入详情页时，浏览历史里没有上级

用户实测反馈："点进去出不来。"

---

## 用户故事

- 用户从 `/lectura` 列表点开一个故事 → 故事详情页顶部显示 `← 阅读`
- 点击 `← 阅读` 回到 `/lectura` 列表
- 课程单元、视频、复习、语法详情同样有对应返回按钮
- 行为是**固定回上级**（不依赖浏览历史），首次直接打开链接也能用

---

## 触动页面与目标路径表

| 当前路径 | 返回按钮文案 | 跳转目标 | 备注 |
|---------|-----------|---------|------|
| `/lectura/[slug]` | `← 阅读` | `/lectura` | **替换**现有 `← 返回 Lectura` 链接（旧链接用 brand 色，文案不一致） |
| `/learn/[slug]` | `← 课程` | `/learn` | |
| `/watch?v=...` | `← 视频` | `/` | 与 SiteNav 中 `/` 这个 tab 的 label "视频" 严格一致 |
| `/vocab/review` | `← 词库` | `/vocab` | |
| `/grammar/[topic]`（如存在） | `← 语法` | `/grammar` | |

文案严格对齐 `SiteNav` 中各 tab 的现有 label，避免顶部导航与返回链接出现两套术语。

**注**：`/vocab`、`/learn`、`/lectura`、`/grammar` 这些**列表页本身不加返回按钮**——它们是顶级导航目的地，SiteHeader 已经提供了所有顶级页面入口。

---

## 设计方案

### 共享组件：`src/app/components/web/BackLink.tsx`

新建一个客户端无关的纯展示组件（Next.js `Link` 即可，不需 `"use client"`）：

```tsx
import Link from "next/link";

type BackLinkProps = {
  href: string;
  label: string;
};

export function BackLink({ href, label }: BackLinkProps) {
  return (
    <Link
      href={href}
      aria-label={`返回${label}`}
      className="mb-2 inline-flex min-h-[44px] items-center gap-1.5 -ml-2 rounded px-2 text-sm font-medium text-gray-600 transition-colors hover:text-gray-900 focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-brand-300"
      data-testid="back-link"
    >
      <span aria-hidden className="text-base leading-none">←</span>
      <span>{label}</span>
    </Link>
  );
}
```

**视觉规格**：
- 字号 `text-sm`（14px），灰色 `text-gray-600`，hover 加深 `text-gray-900`（与 SiteHeader 次级链接对齐）
- 箭头 `←` 字符 + 中文标签，gap 6px
- 触控区 `min-h-[44px]`，满足 iOS HIG 最小可点击区域
- `-ml-2 px-2` 让箭头视觉上与下方标题左对齐（padding 给触控区，负 margin 抵消视觉缩进）
- `mb-2` 内化垂直节奏，避免每个详情页各自调间距
- `focus-visible:ring-2 ring-brand-300` 键盘焦点提示
- `aria-label="返回${label}"` 让屏幕阅读器朗读完整语义（"返回 阅读" 而非孤词"阅读"）
- **不用 brand 色（仅 ring 例外）**——返回是次要动作，避免和"待复习"徽章等主操作抢视觉

### 放置位置

每个详情页在 `<SiteHeader />` 之下、`<header>`（页面标题区）之上插入：

```tsx
<main className="min-h-screen bg-app text-gray-900">
  <SiteHeader />
  <section className="mx-auto max-w-2xl px-4 py-10 sm:px-6">
    <BackLink href="/lectura" label="阅读" />
    <header className="mb-8">
      <h1 className="text-2xl font-bold">{story.title}</h1>
      {/* ... */}
    </header>
    {/* page content */}
  </section>
</main>
```

BackLink 自带 `mb-2`，因此 `<header>` 保持原有 `mb-8` 即可，**不**额外给 `mt-*`。每页一行，5 处分别 hardcode `href` + `label`，不做动态推断（KISS）。

**Lectura 详情页的特殊处理**：现有 `src/app/lectura/[slug]/page.tsx` 已经有一个 `← 返回 Lectura` 链接（hover 色是 `hover:text-brand-600`，与新组件冲突）。Codex1 必须**删除旧链接**，用 `<BackLink href="/lectura" label="阅读" />` 替换，避免页面出现两个返回入口。

### Mobile / Desktop 一致性

- 移动端：BackLink 占一整行，触控舒适
- 桌面端：同样位置，14px 文字不占视觉重量
- 不做特殊响应式分支

### 不做的

- 不实现"浏览器后退"行为（用户已确认要固定回上级）
- 不在列表页加返回按钮（顶级导航目的地）
- 不动 SiteHeader 自身
- 不做面包屑（breadcrumb），就一个返回链接

## 不在本票范围

- 浏览器后退键模拟（方案 Y/Z 都被排除）
- 面包屑导航
- "返回到具体进入路径"（如 watch 页根据 referer 决定回 / 或 /search）
- SiteHeader 自身改造
- 动画过渡
