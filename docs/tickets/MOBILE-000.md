# MOBILE-000 — 移动端地基：查词卡抽屉化 + 设计 token + 导航打磨

**优先级**：P0（移动端重设计 epic 的地基，先于所有单页）
**区域**：`src/app/watch/LookupCard.tsx`（全站共享查词卡）+ 全局移动端 token + `src/app/components/web/MobileNav.tsx` / `SiteHeader.tsx`
**设计**：Gemini1（地基设计稿 `docs/tickets/MOBILE-000-design.md`）
**实现**：Codex1
**测试**：Codex2 + Gemini1 评审 + PM 验收
**预估**：2 天

## 为什么这是"地基"，要先做

移动端重构不是一页页孤立改。有几样东西**几乎每个学习页都在用**，先把它们的移动端形态定死，后续所有页面照用、不重复发明：

1. **查词卡 LookupCard**：全站可点西语文本都弹它（VOCAB-009「全站西语文本统一可点查词」），跨 watch / lectura / talk / learn / grammar。**改一次，全站查词体验一起好。**
2. **移动端设计 token**：触摸目标、字号行距、安全区——设计系统级，定了所有页面统一。
3. **导航/顶栏**：已有 `MobileNav`，随地基一起审查打磨。

> 学习理念背书：词汇靠**阅读输入**自然积累、不靠 SRS 刷卡（产品核心理念）。因此"读/看时点词查词"是全站最高频的核心交互——它的移动端形态值得作为地基优先打磨。

## 目标用户与调性

- 成人 / 学生自学者，**高效克制、专业、不游戏化**（不做儿童早教向）。

---

## 用户可见行为

- 手机上点任意西语文本 → 查词卡从**底部弹出（bottom sheet）**，而非桌面那种卡片：
  - 单手拇指可达、不遮挡正在读的内容、可下滑/点背景关闭。
  - 现有交互全部保留：查词触发视频/朗读暂停与恢复、单词发音、已标记状态、相关短语/用法说明（LEX-003）。
- 全站统一的移动端触摸目标（≥44px）、字号、行距、安全区留白。
- 导航在手机上顺手、跨页一致。
- **桌面端 LookupCard 与各页面不回退。**

## 实施要求

1. **LookupCard 响应式形态**：移动端 = 底部抽屉；桌面端 = 现有卡片。两形态共享同一份查词数据与逻辑，仅呈现容器不同（避免逻辑分叉）。
2. **不破坏调用方**：watch / lectura / talk / learn / grammar 等通过现有 props 调用 LookupCard，改造**保持对外接口兼容**，调用方无需大改。
3. **移动端 token**：在设计系统/全局样式里落地触摸目标、字号阶梯、安全区（`env(safe-area-inset-*)`）等，供后续页面复用。
4. **导航打磨**：审查 `MobileNav` / `SiteHeader` 移动端表现，按设计稿打磨（范围以设计稿为准，不做大改）。

## 待 Gemini1 设计（`docs/tickets/MOBILE-000-design.md`）

- 导航/顶栏移动端形态。

- [ ] 导航/顶栏移动端打磨到位，跨页一致。
- [ ] 全站使用 LookupCard 的页面（watch/lectura/talk/learn/grammar）抽查无回退。
- [ ] 符合 Gemini1 地基设计稿（评审比对）。

- [ ] `npm test` 全绿 + `npm run build` 通过 + `lint:encoding` 通过。
