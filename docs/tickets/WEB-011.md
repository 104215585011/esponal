## 背景

Claude2 评审摘录：
> 全站每个页面对"没有数据"和"出错了"的处理都是各自重写——/vocab 自定义 SVG + 文案、/watch 硬编码"缺少视频参数"、字幕加载失败"暂无字幕"、词查不到"暂不支持该词"、翻译挂了显示原文。视觉、文案语气、操作引导全不一致。

本 ticket 抽出一组复用组件，把全站空/错误态收口。

---

## 一、组件设计

### `<EmptyState>` 单一组件覆盖三类场景

```tsx
type EmptyStateKind = "empty" | "error" | "loading-failed";

type EmptyStateProps = {
  kind: EmptyStateKind;
  title: string;           // 主标题，一句话说清是什么状态
  description?: string;    // 副文，给原因或下一步
  action?: {
    label: string;
    href?: string;         // 外链或站内路由
    onClick?: () => void;  // 客户端按钮
  };
  size?: "sm" | "md" | "lg";  // 默认 md
};
```

视觉：
- `kind="empty"`：灰色书本/笔记图标，文案中性"还没有 XXX"
- `kind="error"`：橙色叹号图标，文案"加载失败，请稍后重试"
- `kind="loading-failed"`：灰色断网图标，文案"暂时无法连接"

样式（用 WEB-009 token）：
- 图标 SVG inline（参考 VocabAccordion 的 EmptyState 实现风格）
- 标题：`text-base font-medium text-gray-700`
- 副文：`text-sm text-gray-400 mt-1.5`
- Action 按钮：`bg-brand-500 hover:bg-brand-600 text-white rounded-card text-sm px-4 py-2 mt-4`
- 容器：`flex flex-col items-center justify-center text-center`
- size 控制 min-height：sm = 160px，md = 240px，lg = 360px
