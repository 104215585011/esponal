## 背景
积分引擎 Phase 1 已实现(数据+扣费+刷新+赠送),但**完全没有用户可见的前端**。积分是新功能,**web 桌面端也需要加**(之前没有任何积分 UI)。本票补齐积分的全部前端表面,web + 移动双端。

## 设计基准
- 遵守 `docs/tickets/MOBILE-design-language.md`(干净现代:白底/Plus Jakarta+Noto Sans SC/翡翠点缀/轻卡片)。
- **会员定价页是旗舰页**:参考用户认可的 **DejaVocab 定价页**(月付/年付/共建者三 tab,卡片式套餐),先出**设计模型**(HTML)给 PM/用户批准,再实现(同首页/课程那套"先看模型"流程)。
- 桌面端不回退;移动端复用外壳(顶栏/底部 tab)。

## 要做的表面(web + 移动都要)

### A. 配额余额展示 + 入口
- **移动端**:顶栏/侧边栏(MOBILE-009 头像侧边栏已有"Esponal 积分"项)显示**当前配额余额**(如 `⚡ 320`)+ 点进账户/会员页。
- **web 桌面端**:站点顶栏导航(SiteNav/SiteHeader)加积分余额 + 会员入口(之前桌面无任何积分 UI,需新增)。
- 余额来源:新增 **`GET /api/credits`**(返回 `{ plan, balanceDisplay, balanceMinor }`,登录用户;调 `getBalanceMinor`)。

### B. 会员/订阅(定价)页 `/membership`(旗舰,出模型)
- 三 tab:**月付 / 年付 / 共建者(终身)**。
- 套餐(数据见下"等级表"):免费 / 进阶 ¥38月·¥365年 / 高阶 ¥48月·¥458年 / 终身进阶 ¥1498 / 终身高阶 ¥1998;标注每月配额、试用、推荐、终身限量。
- **购买按钮文案 = "立即购买"**(接入支付,见 PAY spec);**按购买状态动态显隐/换文案**:
  - 未购买:免费=「当前方案」(禁用),进阶=推荐+立即购买,高阶=立即购买。
  - 已购某付费档:**免费栏消失**;**低于当前档的卡片消失**;当前档 CTA→「点击续费」;更高档 CTA→「点击升级」。
  - 例:购高阶(月)→ 免费+进阶都消失,只剩高阶=续费。
  - 跨周期:购年付→月付 tab 的付费档消失;购共建者(终身)→ 月付+年付 两 tab 内容消失。
  - 推荐角标只在未购买时显示。
- 设计模型:`docs/tickets/CREDITS-membership-mockup.html`(v2,含状态模拟器演示上述逻辑)。
- web + 移动响应式。

### C. 账户/积分页(或并入个人中心)
- 当前等级、配额余额、**用量流水**(CreditTransaction:时间/动作/±配额/余额)。web + 移动。

### D. 积分不足内联提示(依赖 Phase 2)
- 动作扣费返回 ok:false 时,**非模态**内联提示"配额不足 · 升级解锁" + 跳 `/membership`。不弹窗。
- 本票先做 UI 组件 + 接已就绪的扣费返回;Phase 2 接好消费点后联通。
