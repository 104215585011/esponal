## 背景与目标

当前 DB（LEX-CLEANUP-001 后）：427 条 word-kind + 301 条多 token phrase-kind（collocation 125 / phrase 118 / idiom 58）+ 10 条 construction。词汇量停在 A1-A2，B1+ 内容查词时大量 miss → 回落到 DashScope 实时生成（慢、不可控、不入库）。

目标：把 lexicon 扩到 **~12000 词**（覆盖到 B1-C1 高频区），让绝大多数学习材料的查词命中本地库。

> ⚠️ **License 硬约束**：lexicon 后续要做成对外**计费制 API**，所有词表来源必须**允许商业再分发**。优先 CC-BY / MIT / Apache 等宽松许可；**禁止**直接使用受版权保护的成品词表（RAE/CREA、Instituto Cervantes PCIC、Routledge Frequency Dictionary 等）。词频列表只取「词形 + 频次」这类事实数据，不照搬他人的释义/分级文案。

## 管线（沿用 LEX-001 约定：默认 dry-run，`--write` 才落库）

新脚本目录沿用 `scripts/lexicon/`，CSV 中间产物放 `data/`，进度/跳过记录用 JSON（同 seed 脚本约定）。

1. **下载 + 解析词频表** `scripts/lexicon/download-frequency-words.mjs`
   - 拉 FrequencyWords 的 `es` full 列表 → `data/freq-es.txt`（词形 + 频次）
   - 记录来源 URL + commit hash + MIT 许可证声明到 `data/freq-es.LICENSE`（合规留痕，将来 API 要能说清来源）

2. **清洗 + 候选生成** `scripts/lexicon/build-wordlist-candidates.mjs --dry-run/--write`
   - 过滤：纯数字、单字母（保留 y/e/o/u）、专有名词大写残留、明显外来词/拼写噪声
   - lemma 归并（动词变位 → 原型；名词复数 → 单数）
   - **去重**：剔除 DB 已有的 word + phrase（按 lemma，以实际 DB 行为准，约 427 word + 301 phrase）
   - 取频次 top ~15k 输出 `data/wordlist-b1-candidates.csv`（lemma, freq_rank, raw_freq）

   > ⚠️ **2026-05-29 PM 抽检 v1 不通过，step 2 必须返工**（详见下方「PM 抽检结论」）。v1 用了朴素后缀归并，问题结构性：动词变位没还原成原型（está/puedo/quiero 直接当 lemma）、过度截断造非词（gracias→gracia、sabes→sab、entonces→entonce）、凭空造词形（hayes/sólos/deles）、专名和噪声没滤掉。**修正方向（PM 已定）：接成熟西语词形还原库做真正的 lemmatize，不要自己写后缀规则。**

3. **PM 抽检候选** — PM 看 CSV 头部/随机样本，确认噪声率可接受再放行下一步（避免给一堆垃圾词喂 LLM 烧钱）

4. **DeepSeek 逐词补全 + 判级** `scripts/lexicon/seed-b1-words.mjs --dry-run/--write`
   - 复用 `generate-phrase-candidates.mjs` 的 DeepSeek 批处理 + JSON 解析 + 错误隔离/跳过模式（**保留 skip 日志** `data/lexicon-b1-skipped.json`，吸取 LEX-004 教训）
   - 每词让 DeepSeek 产出：`translationZh` / `partOfSpeech` / `cefr`(A1-C2) / `exampleEs` + `exampleZh` / `morphology`
   - **2026-05-29 PM 硬门槛：动词 `forms[]` + `morphology` 必须是真实、可校验的西语变位**。词频表里的变位形只用于发现/去重；入库时仍是一条原型 lemma + 完整 forms/morphology。不得把现有朴素规则生成器当成事实来源继续使用，除非它通过不规则动词 smoke set 校验。
     - 必测 smoke set：`poder` 必含 `puedo/puedes/pude/pudo/pudiendo/podré`；`querer` 必含 `quiero/quieres/quise/querré`；`estar` 必含 `estoy/está/estuvo`。
     - `seed-b1-words.mjs` 对动词应增加校验：生成结果缺关键真实变位时 skip + log，不写入错误 morphology。
   - 取判级为 **A1-C1** 的词入库（C2 暂不要，价值低）。**2026-05-30 PM 决策**：原门槛 B1-C1 导致 pilot 前 50 条全 skip（más/uno/mucho 等高频基础词不在库里但被 DeepSeek 判为 A1/A2），改为 A1-C1 全收，补齐缺口。目标净增 ~12000
   - `--resume` 支持断点续跑（同 seed-a1-a2-words 约定），`--concurrency` 可调

5. **入库后复核**
   - 计数：word-kind 总数应 ≈ 427 + 净增量
   - 抽查例句质量 + 判级合理性（PM 随机抽 30-50 条）
   - 跑 phrase 高亮/lookup 回归，确认没破坏现有体验

## ⚠️ PM 抽检结论 — v1 候选不通过（2026-05-29，返工指令）

Codex1 跑完 step 1-2 产出 `data/wordlist-b1-candidates.csv`（15000 行）。PM 抽检（头部 + 1000/5000 行随机采样）发现 step 2 的 lemma 归并**结构性失败**，闸门**不通过**。好在卡在喂 LLM 之前，未产生成本。

### 实证问题（均为普遍现象，非个例）

1. **动词变位没还原成原型**：`está/estoy/puedo/puede/quiero/hizo/creo/sé/he/debe/ven/prefiero/piden/juegan/hemos/llevaré/encantaría/oiga` 等大量变位形直接当 lemma 收进来。它们本应收敛到 estar/poder/querer/… 这些原型，而原型多数已在 733 词库 → 去重也连带失效。
2. **过度截断造非词**：`gracias→gracia`、`sabes→sab`、`entonces→entonce`。垃圾 stem 喂 LLM 纯浪费。
3. **凭空捏造词形**：source_forms 里出现 `hayes/sólos/deles/pises` 这类不存在的复数。
4. **噪声未滤**：139 条 ≤2 字符 token（oh/ti/sr/eh/ah/ok/dr/ud/uh/um/fe/ei…）；专名 `johnny/patty/nikki`；旧正字法 `sólo`(应 solo)/`fuí`(应 fui)。

### 返工要求（step 2 重做）

1. **lemma 还原改用成熟西语词形还原库**（PM 已拍板），不要再自己写后缀规则。推荐：
   - **simplemma**（Python，**MIT**，纯词典查表、无需下大模型、轻量）— 首选
   - 或 spaCy `es_core_news_*`（代码 MIT；模型走生成期工具用途）
   - **License 说明**：lemmatizer 是**生成期工具**，不随 API 再分发；产出的「变位形→原型」是事实性数据，不可版权化 → 用它不污染我们的商用许可。本项目是 Node，一次性管线脚本可 shell 调 Python（同 feature_list 编辑已有先例）。
2. 还原后再去重：变位形先归原型，**再**与 DB 现有 733 word / 301 phrase 比对剔除（修复 v1 的去重失效）。
3. 噪声过滤补齐：≤2 字符 token（除 y/e/o/u 等合法虚词）、专有名词（首字母大写且不在普通词典）、旧正字法归一（sólo→solo、guión→guion 等）、明显拼写错误/外来人名。
4. 重出候选 CSV 后 **PM 再抽检一次**（step 3 二轮）才放行 step 4。
5. 加个轻量自检：脚本跑完打印「候选数 / 被还原合并掉的变位形数 / 因已在库去重掉的数 / 被噪声过滤掉的数」，方便 PM 一眼看出归并是否真的生效。

**⚠️ 阻塞 step 4 — lemmatizer 复现性 bug（PM 2026-05-29 已定位，Codex1 启动 step 4 前必须先修）**

PM 在自己环境跑 `build-wordlist-candidates.mjs` dry-run 崩溃，已定位根因（数据本身没问题，装上 simplemma 即可复现 CSV；坏的是依赖固化 + 错误处理）：

- **依赖未声明**：lemmatizer 是 `spawn("python", ["scripts/lexicon/simplemma_lemmatize.py"])`，py 脚本裸 `import simplemma`。但仓库**无 requirements.txt / pyproject**，simplemma 未被固化。PM 环境实测 `ModuleNotFoundError: No module named 'simplemma'`。换环境/全新 checkout 必跑不动。
- **错误处理脆（Node 真 bug）**：`runPythonLemmatizer` 有 `child.on("error")` 但**缺 `child.stdin.on("error")`**。子进程 import 失败秒退时，`child.stdin.end(JSON)` 写已关闭管道 → 未处理 socket error → 整个 Node 崩，且吞掉真正的 python stderr，只剩看不懂的 `write EOF`。

  **修复清单（Codex1）**：
  1. 加 `scripts/lexicon/requirements.txt`（pin `simplemma==<版本>`），README/ticket 注明 `pip install -r scripts/lexicon/requirements.txt`
  2. `runPythonLemmatizer` 加 `child.stdin.on("error", …)` 处理 EPIPE；子进程非 0 退出时把 **stderr 原样抛出**（让 `ModuleNotFoundError` 浮上来），报错信息带可操作指引（提示装依赖）
  3. 启动前做 1 词探活（preflight），失败 fast-fail，不要灌 15000 词才崩
  4. 修完 Codex1 在干净环境复跑一遍 dry-run，确认能完整产出 stats 行 + CSV 可复现，再开 step 4
