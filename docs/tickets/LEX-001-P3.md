## 背景

Phase 2 已经把 A1-A2 单词种子跑通（动词 / 名词 / 形容词三条路径都对）。Phase 3 是产品差异化的关键——**固定搭配（colocaciones）**——西语助手最有价值的部分，公开词典基本没覆盖。

> 用户原话："平常背单词都只是单独的背，但实际上如果能够记住一些固定搭配的词语就能更好的口语或者使用。"

---

## 范围

约 **500 条** A1-A2 高频固定搭配，按 `kind` 分三类：

| kind | 例子 | 数量目标 |
|---|---|---|
| `collocation` | `tener que` / `ir a` / `hay que` / `acabar de` / `estar + gerundio` | 200 条 |
| `phrase` | `buenos días` / `por favor` / `hasta luego` / `de nada` | 200 条 |
| `idiom` | `no hay de qué` / `valer la pena` | 100 条 |

## 实施流程

### 3.1 LLM 生成候选 CSV（Codex1）

新建 `scripts/lexicon/generate-phrase-candidates.mjs`：

- 调 DeepSeek V3，分三轮 prompt：
  1. "列 200 个 A1-A2 西语动词性固定搭配（colocaciones），每条返回 lemma / kind=collocation / level / 中文释义 / 类别（verb_periphrasis / verb_object 等）"
  2. "列 200 个 A1-A2 西语常用问候 / 礼貌 / 时间地点短语（phrases）"
  3. "列 100 个 A1-A2 常见习语（idiom），但只挑学习者真正用得到的，不要太书面"
- 输出 `data/phrases-a1-a2-candidates.csv`，PM 在 Excel/Numbers 里删改增

CSV 格式：
```csv
lemma,kind,level,category,translation_zh,keep
tener que,collocation,A1,verb_periphrasis,必须/不得不,1
ir a,collocation,A1,verb_periphrasis,将要,1
acabar de,collocation,A2,verb_periphrasis,刚刚,1
por favor,phrase,A1,courtesy,请,1
buenos días,phrase,A1,greeting,早上好,1
...
```

`keep` 字段：PM 改成 `0` 的行会被 seed 脚本跳过。

### 3.2 PM 审核（Claude1）

- 用 Excel / Numbers 打开 CSV
- 删改增，约 1-2 小时
- 删除模型生成的低质量条目（机翻味重、过于书面、超出 A1-A2 范围等）
- 把过审的最终版保存为 `data/phrases-a1-a2-seed.csv`

### 3.3 短语种子脚本（Codex1）

新建 `scripts/lexicon/seed-a1-a2-phrases.mjs`：

输入：`data/phrases-a1-a2-seed.csv`

对每条短语：
1. **Tatoeba 例句匹配**（关键）：
   - 在 `data/tatoeba-es-zh.jsonl` 里搜含完整短语字符串（unicode normalize + lowercase）的 ES-ZH 对
   - 找不到 → 调 DeepSeek 让它生成 2-3 条自然例句（标 `source: "llm-generated"`）
   - 取 2-3 条
2. **释义增强**：DeepSeek 在 CSV 已有 `translation_zh` 基础上，生成：
   - `explanationZh`（用法说明，例如 "tener que + 不定式 → 表达必须"）
   - `translationEn`
   - 如果 collocation 是动词短语，提供常见时态搭配（"通常配现在时和过去时"）
3. **写入 LexiconEntry**：
   - `kind`: collocation / phrase / idiom
   - `lemma`: CSV 提供
   - `displayForm`: 保留首字母大小写 / 标点
   - `forms`: `[lemma]`（短语本身不变形）
   - `morphology`: null（短语没形态变化）
   - `partOfSpeech`: `"collocation_verb"` / `"phrase_greeting"` 等基于 category 拼接
   - `sources`: `["tatoeba", "llm-deepseek"]` 或 `["llm-deepseek"]`（依例句来源）
   - `licenseCode`: 同 phase 2

CLI 参数沿用 Phase 2 风格：
- `--write`（默认 dry-run）
- `--limit N`
- `--resume`
- `--concurrency N`
- `--csv PATH`（指定输入 CSV）
- `--help`

### 3.4 反查注入（重要）

短语的 lemma 是多 token（如 `tener que`），lookup 时用户点击 `tener` 或 `que` 单字不会命中短语条目。需要：

- 在 LexiconEntry 表新加索引或在 lookup 路径里做：**短语的所有 token 都触发"可能短语提示"**
- 简化方案：lookup 单词时同时模糊查 LexiconEntry 里 lemma 包含该 token 的短语，作为附加结果返回
- 这个反查逻辑放到 Phase 4（lookup API 接入）做，本 ticket 只保证数据正确入库
