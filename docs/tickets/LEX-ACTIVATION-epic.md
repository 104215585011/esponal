# Epic — 语料库活化(AI 挖掘平台语料库)

**状态**：规划(下一波;移动端 epic + CORPUS-001 收尾后启动)
**战略来源**：memory `ai-corpus-mining`
**PM 排期**：Claude1 2026-06-03

> 把后台平台语料库(LEX 词典/短语库)从**人工预建的静态库**,变成**在用户使用中、靠 AI 辅助、带质量闸、会自己长大的活语料库**。
> 区分:本 epic = **平台共享语料库**;CORPUS-001 = 用户个人收藏(两回事)。

## 贯穿原则(每张票都遵守 — 质量闸)
- **确定性字段不用 AI**:变位/性数用规则引擎/词典(LEX-005 假变位教训)。AI 只碰软字段(中文释义/用法)。
- **例句用真实语料**(Tatoeba/真实内容),不让 AI 编。
- **交叉校验**:AI 释义对照权威词典/第二个模型,分歧标记。
- **置信度 + 审核闸**:高置信/高频自动入候选,低置信进待审。
- **金库 vs 候选库分层**:license-clean 精库不被 AI 直接写;AI 挖的进候选层(标记),审核升级。
- **用户纠错**:词条加报错入口,使用量暴露错误。

## 票序列(按价值/依赖排)

### LEX-007 — 查词缺口回填 + 质量闸 MVP 【Phase 1·先做·最高杠杆】
- **目标**:验证整条「挖掘→校验→沉淀」管线,并直接削减 DashScope 回落成本。
- 范围:① 记录"本地未命中、回落 AI"的查词(缺口词 + 频次);② 缺口词由 AI 生成软字段 + 规则引擎补变位/性数 + 真语料配例句;③ 交叉校验 + 置信度打分;④ 高置信/高频自动入**候选库层**(下次本地命中、免费),低置信进待审队列(stub 即可)。
- 产出价值:查得越多、库越全、回落 AI 越少(对接积分制度毛利)。
- **⚠️ 质量关键,build 前必须先 brainstorm**(用 superpowers:brainstorming 把校验/置信/入库规则定清楚)。

### LEX-008 — 候选库审核队列 + 升级金库 + 用户纠错
- 候选层数据模型 + PM/admin **审核界面**(看 AI 生成,批准→升级进金库 / 拒绝)。
- 前端**报错/纠正**入口 + 处理流(被多次报错的词条优先复审)。

### LEX-009 — 内容短语挖掘
- 从用户消费的字幕/文章,用 AI 抽**固定搭配/习语 + 真实例句** → 短语库候选(PHRASE-001/LEX-003 体系)→ 走同一审核闸。

### LEX-010 — 使用数据校准难度/频率
- 用"遇到/查/标记"等使用数据,校准分级(A1/A2/B1)与高频排序,优于静态频率表。

## 依赖 / 节奏
