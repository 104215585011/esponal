### 3. Service Worker

不引入 `next-pwa` 老牌包（Next 14 兼容性差）。用现代方案：**Workbox 手写**，或更简单——**用 next-pwa 的轻量替代 `@serwist/next`**。

推荐 **`@serwist/next`**：
- 由 next-pwa 原作者维护，针对 Next 13+ 的 App Router 设计
- 配置最简

```bash
npm install @serwist/next serwist
```

`next.config.mjs` 包一层：
```ts
import withSerwistInit from "@serwist/next";

const withSerwist = withSerwistInit({
  swSrc: "src/sw.ts",
  swDest: "public/sw.js",
  cacheOnNavigation: true,
  reloadOnOnline: true
});

export default withSentryConfig(withSerwist(nextConfig), { ... });
```

新建 `src/sw.ts`：
```ts
import { defaultCache } from "@serwist/next/worker";
import { Serwist } from "serwist";

declare const self: ServiceWorkerGlobalScope;

const serwist = new Serwist({
  precacheEntries: self.__SW_MANIFEST,
  skipWaiting: true,
  clientsClaim: true,
  navigationPreload: true,
  runtimeCaching: defaultCache
});

serwist.addEventListeners();
```

### 4. 缓存策略

不同资源走不同策略：

| 资源 | 策略 | 理由 |
|---|---|---|
| 静态 JS/CSS/字体 | CacheFirst | Next.js 文件名带 hash，更新即失效 |
| `/lectura/*` 页面 HTML | StaleWhileRevalidate | 离线时立刻显示旧版，后台刷新 |
| `/api/subtitle/*` 翻译结果 | StaleWhileRevalidate（7 天） | 字幕通常不变 |
| `/api/vocab/*` | NetworkOnly | 总是要最新 |
| `/api/auth/*` | NetworkOnly | 安全相关绝不缓存 |
| YouTube iframe / 视频流 | NetworkOnly | 跨域、大、不缓存 |
| 应用图标 / manifest | CacheFirst | 几乎不变 |

`defaultCache` 已经处理大部分情况，按需在 `runtimeCaching` 加自定义规则。

### 5. 「添加到主屏」引导

不要做 push 弹窗骚扰用户。简单做法：

- 在 `SiteHeader` 或 `HomeHero` 加一个不显眼的「📱 添加到主屏」按钮（只在移动端 + 没装过时显示）
- 点击触发 `beforeinstallprompt` 事件（已经在 Window 里捕获并存住）
- 用户点了就走系统装机流程

实现细节：
- 新建 `src/app/components/web/InstallPrompt.tsx` 客户端组件
- 监听 `beforeinstallprompt` → preventDefault + 存事件
- 渲染按钮（仅事件可用 + 非 standalone 时显示）
- 点按钮 → `event.prompt()` → 用户选择

挂在哪？建议 `HomeHero` 底部一行小字（「也可以添加到主屏，离线读 Lectura →」）。

### 6. 离线 fallback 页

新建 `src/app/offline/page.tsx`：

```tsx
export default function OfflinePage() {
  return (
    <main className="bg-app min-h-screen px-6 py-20 text-center">

```

## 六、注意

- Service Worker 一旦上线很难撤回（用户浏览器会一直跑旧版直到刷新）。第一次上线**先用 `skipWaiting: false` 测试**，确认 OK 后再开 `true`
- Workbox / Serwist 在 dev 模式下默认禁用（避免热更新混乱），本地测试要 `npm run build && npm run start` 才能验
- Vercel 部署 PWA 几乎零配置（已经支持 manifest + sw 静态文件）
- 图标 maskable 版的「安全区」**必须居中 80%**，否则 Android 上会裁掉部分内容
- `beforeinstallprompt` 事件**只在 PWA 标准达标时触发**——manifest、sw、HTTPS、图标四件齐全才会出
