# LEX-001 — 自有语料库基础设施 + A1-A2 试点

**类型**：后端基础设施 + 数据工程
**实现**：Codex1
**测试**：Codex2
**最终验收**：Claude1
**前置依赖**：无
**预估**：7-10 天（分四阶段）
**优先级**：高 — 产品护城河，但试点先小规模

---

## 背景与战略

目前查词依赖外部 API（DeepSeek（deepseek-chat / V3） + Microsoft 翻译），每次都耗时 + 短语几乎无效。

**战略目标**：自建语料库 `LexiconEntry`，成为产品长期护城河。**未来计划包装成计费制公开 API** 开放给其他西语学习产品使用，因此**所有数据源必须 license 兼容商用再分发**。

**试点范围**：A1-A2，约 **3000 单词 + 500 固定搭配**。先小规模验证流程跑通，再考虑扩到 B1+。

**关键设计原则**：
1. 数据源 license 清晰可商用（CC-BY 优先，避免 SA 传染）
2. 抓取脚本本地运行，可断点续传，PM 长期挂机执行
3. 中文翻译优先 Tatoeba，缺失部分用 LLM 兜底
4. 每条记录追踪来源 + 质量分

---

## LLM Provider 决策

**所有批量 LLM 调用（种子翻译、释义生成、固定搭配解释）走 DeepSeek（deepseek-chat / V3）。**

- 复用 `src/lib/talk/model-client.ts` 的 DeepSeek 客户端抽象，或在 `src/lib/lexicon/llm-client.ts` 新建一份带批量并发控制的 wrapper
- 环境变量直接用现有的 `DEEPSEEK_API_KEY` / `DEEPSEEK_BASE_URL` / `DEEPSEEK_MODEL`
- 为何选 DeepSeek：纯文本任务足够强，价格约为 DashScope 的 1/5
- **lookup 兜底链路保留 DashScope**（已稳定运行，VOCAB-004 路径不动）

预算估算：3500 词条 × 平均 500 token ≈ 175 万 token，DeepSeek 全跑下来约 **2-3 元 RMB**。

## 形态覆盖要求（关键设计原则）

**西语形态变化非常多，光收 lemma 不够，必须收齐变体形态，才能让用户点 `hablaba` 直接回到 `hablar`。**

### 动词（最复杂）

每个动词必须收齐以下形态。当前 `src/lib/conjugate.ts` 的 `tryConjugateVerb` 已覆盖前 6 项，**需要扩展**：

| 时态 | 已有 | A1-A2 必备 |
|---|---|---|
| Presente | ✅ | ✅ |
| Pretérito indefinido | ✅ | ✅ |
| Pretérito imperfecto | ✅ | ✅ |
| Futuro simple | ✅ | ✅ |
| Condicional simple | ✅ | A2+ |
| Presente subjuntivo | ✅ | A2 入门 |
| Imperativo | ✅ | ✅ |
| **Pretérito perfecto compuesto**（he hablado） | ❌ 待加 | **A1 必备** |
| **Participio**（hablado） | ❌ 待加 | **A1 必备**（构成完成时） |
| **Gerundio**（hablando） | ❌ 待加 | **A1 必备**（构成进行时） |
| Pretérito pluscuamperfecto | ❌ | B1 可选 |
| Futuro / Condicional compuesto | ❌ | B1 可选 |
| Imperfecto subjuntivo | ❌ | B1 可选 |

**实施**：
- 扩展 `tryConjugateVerb` 返回值，加上 `participio` / `gerundio` / `preteritoPerfecto`（用 haber 现在时 + participio 组合）
- LexiconEntry.`forms` 要展平所有变位形态进数组，便于反查匹配
- LexiconEntry.examples 旁边附 `conjugations` JSON（结构化时态表，便于前端展示）

### 名词

每个名词必须含：
- `lemma`：单数原形（如 `casa`）
- `forms`：[lemma, plural]（如 `["casa", "casas"]`）
- `partOfSpeech`：`noun_m` / `noun_f` / `noun_mf`（性别两可）
- 如有不规则复数（如 `lápiz → lápices`），手工标注

### 形容词

每个形容词必须含 4 形态：
- `forms`：[masc_sg, masc_pl, fem_sg, fem_pl]
- 例 `bueno` → `["bueno", "buenos", "buena", "buenas"]`
- 不变形容词（如 `azul`、`feliz`）只有 2 形态：[sg, pl]

### 反查匹配（核心用户场景）

用户在阅读时点 `hablaba` → 查词 API：
```
1. 在 LexiconEntry.forms 数组里全表搜 "hablaba"
2. 命中 → 返回 lemma="hablar" 的完整条目
3. 同时返回 "当前形态是 imperfecto 第 1/3 人称单数"
4. 用户体验：看到 hablar 的释义 + 这个形态的位置
```

为此 `forms` 数组必须包含 lemma 之外的**所有变体**。

## 阶段拆分

### Phase 1：Schema + 基础设施（1-2 天）

#### 1.1 新增 Prisma 表 `LexiconEntry`

```prisma
model LexiconEntry {
  id            String          @id @default(cuid())
  kind          LexiconKind     // word | phrase | collocation | idiom
  lemma         String          // 西语原形，归一化为小写
  displayForm   String          // 展示原文（保留大小写、变音符号）
  forms         String[]        // 词形变化（动词变位等）
  partOfSpeech  String?         // noun_m / noun_f / verb / adj / adv / phrase_verb / ...
  level         CefrLevel?      // A1 / A2 / B1 / ...
  frequency     Int?            // 语料频次（越大越常用）
  ipa           String?         // 国际音标
  audioUrl      String?         // 发音 mp3

  // 翻译与解释
  translationZh String?         // 中文主译
  translationEn String?         // 英文主译（便于校对）
  explanationZh String?         // 用法 / 语法解释

  // 例句
  examples      Json?           // [{ es, zh, en?, source }]

  // 形态结构化（便于前端展示完整变位 / 性数表）
  morphology    Json?           // 动词：{ presente:{yo,tu,...}, imperfecto:{...}, ... }
                                // 名词：{ singular:"casa", plural:"casas" }
                                // 形容词：{ masc_sg:"bueno", masc_pl:..., fem_sg:..., fem_pl:... }

  // 关联
  collocations  String[]        // 关联固定搭配 lemma 列表

  // 元数据
  sources       String[]        // ["tatoeba", "wiktionary", "llm-deepseek"]
  licenseCode   String          // "CC-BY-2.0-FR" / "CC-BY-SA-3.0" / "LLM-derived" / "internal"
  qualityScore  Int             @default(0) // 0-5，人工校对过=5
  lookupCount   Int             @default(0) // 用户查询次数
  
  createdAt     DateTime        @default(now())
  updatedAt     DateTime        @updatedAt

  @@unique([kind, lemma])
  @@index([level])
  @@index([frequency])
  @@index([lookupCount])
}

enum LexiconKind {
  word
  phrase
  collocation
  idiom
}

enum CefrLevel {
  A1
  A2
  B1
  B2
  C1
  C2
}
```

#### 1.2 Migration
```
prisma migrate dev --name add_lexicon_entry
```

#### 1.3 基础 lib `src/lib/lexicon.ts`

### Phase 2：Tatoeba 数据导入脚本（2-3 天）

#### 2.1 下载 Tatoeba dumps（在 PM 本机执行）

数据源：https://tatoeba.org/en/downloads

需要的文件：
- `sentences.csv`（全语言句子，约 1.2GB）
- `links.csv`（翻译对应关系）

脚本：`scripts/lexicon/download-tatoeba.mjs`
- 从 Tatoeba 官方下载页拉取 dump
- 解压到 `data/tatoeba/`
- 校验文件完整性（行数、checksum）

⚠️ PM 本机预留磁盘 5GB

#### 2.2 解析与配对 `scripts/lexicon/parse-tatoeba.mjs`

- 读 `sentences.csv` → 抽出 lang=spa 的句子（约 750k 条）和 lang=cmn 的句子（约 100k 条）
- 读 `links.csv` → 构建 ES↔ZH 配对索引
- 输出：`data/tatoeba-es-zh.jsonl`，每行一条 `{ es, zh, esId, zhId }`

预期产出：**约 5-10 万条 ES-ZH 平行句对**

#### 2.3 扩展 `tryConjugateVerb`（前置）

在跑种子脚本前，先扩展 `src/lib/conjugate.ts`：
- 加 `participio`（用 `spanish-verbs` 的 `INDICATIVE_PERFECT` 或手动规则）
- 加 `gerundio`
- 加 `preteritoPerfectoCompuesto`：构造方式 = haber 现在时变位 + participio
- 增量单元测试覆盖 5 个典型动词（含规则 + 不规则：hablar / comer / vivir / ser / tener）

#### 2.4 抽取 A1-A2 单词种子

`scripts/lexicon/seed-a1-a2-words.mjs`
- 输入：A1-A2 单词列表（来源：PM 提供 CSV，或从我们现有 `foundation.ts` 课程内容 + Tatoeba 高频词交叉得到）
- 对每个 lemma：
  1. 调 `tryConjugateVerb`（动词）或手工查表（名词性别 + 复数、形容词 4 形态）→ 填 `forms` 数组 + `morphology` JSON
  2. 在 Tatoeba ES-ZH pairs 中找 1-3 条含该词的例句
  3. 调 DeepSeek（deepseek-chat / V3） 生成中文释义 + 词性 + 简单用法说明
  4. 写入 `LexiconEntry`，`sources: ["tatoeba", "llm-deepseek"]`

**性别 / 复数判定逻辑**：
- 名词：调 DeepSeek（deepseek-chat / V3） 让它返回 `{ gender: "m"|"f"|"mf", plural: "casas" }`
- 形容词：DeepSeek（deepseek-chat / V3） 返回 4 形态（让模型校验自洽，如 `bueno → buenos/buena/buenas`）
- 不规则形态优先信任 DeepSeek（deepseek-chat / V3），但 PM 抽检时若发现错误要回填校正

PM 控制：
- 脚本带 `--limit N` 参数，可分批跑（如先跑 100 条人工抽检，再放开）
- 进度持久化到 `data/lexicon-progress.json`，可断点续传

### Phase 3：A1-A2 固定搭配种子（2-3 天）

固定搭配是产品差异化重点。

#### 3.1 PM 提供初始 500 条候选清单

格式 `data/phrases-a1-a2-seed.csv`：
```
lemma,kind,level,category
tener que,collocation,A1,verb_periphrasis
ir a,collocation,A1,verb_periphrasis
hay que,collocation,A1,verb_periphrasis
acabar de,collocation,A2,verb_periphrasis
por favor,phrase,A1,courtesy
...
```

> **来源建议**：
> 1. LLM 出第一稿（让 DeepSeek（deepseek-chat / V3） 列 A1-A2 高频固定搭配 500 条）
> 2. PM 在 spreadsheet 里删改增（约 1-2 小时）
> 3. 落到 CSV 作为种子

#### 3.2 `scripts/lexicon/seed-phrases.mjs`

对每条短语：
1. 在 Tatoeba ES-ZH 里搜含该短语的句子（fuzzy match）
2. LLM 生成「中文释义 + 用法说明 + 2-3 条例句」
3. 写入 `LexiconEntry`，kind=`collocation` / `phrase`

---

### Phase 4：Lookup API 接入语料库（1-2 天）

#### 4.1 改造 `src/app/api/vocab/lookup/route.ts`

新查词流程：
```
1. 查询 LexiconEntry（lemma 精确匹配）
   → 命中 → 返回，lookupCount +1，记录耗时（应 < 50ms）
2. 未命中 → 调外部 API（DeepSeek（deepseek-chat / V3） + Microsoft）
   → 返回结果给用户的同时，**异步** upsert LexiconEntry，sources: ["llm-deepseek"], qualityScore: 1
3. 用户下次查同样的词 → 走步骤 1
```

#### 4.2 监控指标

在响应头加：
- `X-Lexicon-Hit: 1 | 0`（命中本地）
- `X-Lookup-Source: lexicon | external`
- `X-Lookup-Latency-Ms: ...`

PM 可以观察缓存命中率走势。

## 风险与待澄清

1. **Tatoeba 数据规模**：实际 ES-ZH pairs 数量不确定。如果远低于 5 万，Phase 2 抽取例句的命中率会差，需要 LLM 兜底更多。
2. **DeepSeek（deepseek-chat / V3） 调用成本**：批量调用 3000 词 + 500 短语 = 约 3500 次 LLM 请求。需 PM 确认 API 余额。
3. **license 验证**：Tatoeba CC-BY-2.0-FR 商用 OK，但二次分发需署名。**未来开放 API 时需保留 source 字段**让用户看得到归属。
