# LEX-001 Phase 4 — Lookup API 接入 LexiconEntry

**类型**：后端
**实现**：Codex1
**测试**：Codex2 + Claude1 实测
**前置依赖**：Phase 1-3 已通过（DB 现有 795 条 LexiconEntry）
**预估**：1-2 天
**优先级**：高（整个 LEX-001 试点的回报点）

---

## 背景

LEX-001 Phase 1-3 已经把 A1-A2 单词 + 固定搭配的本地语料库建好（795 条）。但**用户查词时还在走外部 API**（DashScope + 微软翻译），没有任何本地命中。

Phase 4 就是把本地库接到 `/api/vocab/lookup`，让常用词查询从外部 API → **本地毫秒级返回**。

---

## 实施

### 4.1 改造 `src/app/api/vocab/lookup/route.ts`

**新查词流程**：

```
1. 收到 lookup(lemma) 请求
2. 在 LexiconEntry 里精确匹配（kind=word 优先，命中则返回）
3. 同时在 LexiconEntry.forms 数组里搜（处理变位形态，如 hablaba → hablar）
4. 命中本地 → 返回 + lookupCount +1，response header X-Lexicon-Hit:1
5. 未命中 → 调外部 API（现有 DashScope 路径不动）
6. 外部 API 返回后：
   - 给用户返回
   - **异步** upsert LexiconEntry，sources=["llm-deepseek"]，qualityScore=1
   - （这样未来同样的词查询走本地）
```

### 4.2 LexiconEntry 返回结构 → 现有 LookupCard 兼容

当前 LookupCard 显示这些字段：lemma / partOfSpeech / translation / examples / conjugations / saved 状态。

LexiconEntry 已有所有这些数据（morphology JSON 含 conjugations）。**关键约束**：返回 payload 的字段名/结构必须和现有 `lookup` 路由一致，前端零改动。

如果字段名不同，在 route.ts 里做一层 mapper。

### 4.3 短语反查（关键体验）

短语 lemma 是多 token（`tener que`）。用户点击 `tener` 单字查词时，**附带返回**包含该 token 的短语条目作为 `relatedPhrases`。

实现：
```sql
-- 当用户查 lemma="tener" 时，额外查
SELECT * FROM LexiconEntry
WHERE kind IN ('collocation','phrase','idiom')
  AND lemma LIKE 'tener %' OR lemma LIKE '% tener %' OR lemma LIKE '% tener'
LIMIT 5
```

response 里加：
```ts
{
  ...existing word lookup payload,
  relatedPhrases: [
    { lemma: "tener que", translationZh: "必须", kind: "collocation" },
    { lemma: "tener ganas de", translationZh: "想做某事（+不定式）", kind: "collocation" },
    ...
  ]
}
```

前端 LookupCard 在 word 释义底下加一个折叠区「相关短语」展示。**前端这部分改造可以作为独立的 UI 子 ticket** 给 Gemini1 设计；本 ticket 只保证 API 返回 relatedPhrases 字段。

### 4.4 响应头监控

加 response header：
- `X-Lexicon-Hit: 1 | 0`（命中本地）
- `X-Lookup-Source: lexicon | external`
- `X-Lookup-Latency-Ms: <int>`

PM 可以通过 DevTools 观察命中率走势。

### 4.5 异步 upsert

外部 API 返回结果后，**不能阻塞响应**就直接 upsert。建议：

```ts
// 给用户返回结果
const response = NextResponse.json(payload);

// 后台异步写
setImmediate(async () => {
  try {
    await upsertLexiconEntry({ lemma, translation_zh, ..., sources: ["llm-deepseek"], qualityScore: 1 });
  } catch (err) {
    console.warn('Lexicon backfill failed:', err.message);
  }
});

return response;
```
