# PHON-002 — 元音 / 辅音基础介绍模块

**优先级**：P2（PHON-003 / PHON-004 的前置知识）
**区域**：phonics
**依赖**：PHON-001（字母表页面已存在）

---

## 背景

现在 `/phonics` 直接展示 27 个字母格，没有任何基础铺垫。
中文母语者需要先理解"元音固定发音"和"强/弱元音"的概念，
才能正确处理后续的条件变音（PHON-003）和重音规则（PHON-004）。

## 范围

### 做

在字母表格**上方**增加一个「发音基础」介绍块，包含三段：

**段 1 — 元音（Vocales）**
- 列出 5 个元音：a / e / i / o / u，每个可点击播放单音 TTS
- 说明：西语元音发音固定，每个字母只有一种读法（区别于英语）
- 中文注音对照：阿 / 诶 / 一 / 哦 / 乌

**段 2 — 强元音 vs 弱元音**
- 强元音（Vocales fuertes）：a / e / o — 可独立撑起一个音节
- 弱元音（Vocales débiles）：i / u — 紧挨强元音时"让位"，合并为同一音节
- 每类给 1～2 个可点击的例词（播整词）

**段 3 — 二合元音（Diptongo）**
- 强 + 弱 / 弱 + 强 / 弱 + 弱 在同一音节内合并
- 3 个例词，合并的音节部分在文字上高亮（不同色或下划线），可点击播音：
  - **bue**no（ue = 弱+强）
  - ciu**dad**（iu = 弱+弱）
  - **ai**re（ai = 强+弱）

**辅音（Consonantes）**
- 一句话带过：其余 22 个字母均为辅音，发音依上下文变化（见各字母详情）
- 不需要独立模块，放在元音介绍块结尾一行即可

### 不做
- ❌ 三合元音（Triptongo）——A1 阶段过难
- ❌ 双元音的完整规则表格——只给例子，不堆规则

## UI 要求

- 介绍块放在字母格**上方**，与字母格之间有视觉分隔（`border-b` 或 section gap）
- 每个可点击的音 / 词用统一的小播放按钮样式（与现有字母卡一致）
- 高亮二合元音用 `text-brand-600 font-semibold` 或下划线，不要 badge

## 音频方案

使用**预生成**音频（与 PHON-001 一致）：
- 单音（a/e/i/o/u）：已有，复用 `/audio/phonics/letters/*.mp3`
- 例词（bueno / ciudad / aire）：新增，由 `scripts/generate-phonics-audio.mjs` 生成，
  放入 `/audio/phonics/words/`
