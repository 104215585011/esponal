# LEX-003 — LookupCard 显示「相关短语」+ construction 用法说明

**类型**：前端（UI 设计 + 实现）
**设计**：Gemini1
**实现**：Codex1
**测试**：Codex2 + PM 实测
**前置依赖**：LEX-001 P4 已通过（API 已返回 `relatedPhrases`）、PHRASE-001 已通过（叠卡 `LookupCardStack` 已存在）
**预估**：1-1.5 天
**优先级**：中（让 P4 攒下来的短语反查数据对用户可见 + 可点）

---

## 背景与价值

LEX-001 Phase 4 已让 `/api/vocab/lookup` 在查单词时**附带返回** `relatedPhrases`——包含该词的固定搭配（如查 `tener` 返回 `tener que` / `tener ganas de`）。但**前端 LookupCard 完全没展示这个字段**，数据白攒了。

LEX-003 把它呈现出来：用户查一个动词/名词时，词卡底部出现一个轻量「相关短语」区，点击某条短语 → 用现有 `LookupCardStack` 叠一张短语卡（复用 PHRASE-001 的叠卡机制，最深 2 层）。

附带任务：LEX-CLEANUP-001 引入了 `kind=construction`（gustar 倒装、soler+inf 这类单动词特殊构式），API 在这些词上返回 `usageNote`，词卡也应展示。

---

## 后端现状（已就绪，本 ticket 不改后端）

`/api/vocab/lookup` 返回 payload 里已有：

```ts
relatedPhrases: { lemma: string; translationZh: string; kind: "collocation" | "phrase" | "idiom" }[]  // 最多 5 条
usageNote?: string | null   // construction kind 的用法说明（如 gustar 倒装结构提示）
```

> 实现位置参考：`src/app/api/vocab/lookup/route.ts`（`mapLexiconEntryToLookupPayload` + `findRelatedPhraseEntries` + `findConstructionEntry`）。**本 ticket 只动前端**，若发现字段缺失再回头补后端。

---

## 用户体验目标

查 `tener` 时：

1. 词卡正常显示释义 + 例句（行为不变）
2. 例句下方、「加入我的词库」按钮上方，出现一个轻量分区「相关短语」
3. 列出最多 5 条：`tener que · 必须`、`tener ganas de · 想做某事（+不定式）` …，每条带 `[固定搭配]`/`[短语]`/`[习语]` 小 badge（沿用 PHRASE-001 词卡的 amber badge 体系）
4. 点击某条 → 用 `LookupCardStack` 叠一张**短语 lookup card**（lookupKind="phrase"），受 2 层上限约束
5. construction 词（如 `gustar`）额外在释义区下方显示 `usageNote` 用法说明（轻量提示样式，不喧宾夺主）

无 `relatedPhrases` 时整个分区不渲染（不留空 div）。

## 段 2：前端实现（Codex1，blocked by 段 1）

#### 2.1 扩展 `LookupResponse` 类型（`src/app/watch/LookupCard.tsx`）

```ts
type RelatedPhrase = { lemma: string; translationZh: string; kind: "collocation" | "phrase" | "idiom" };

type LookupResponse = {
  // ...existing fields
  relatedPhrases?: RelatedPhrase[];
  usageNote?: string | null;
};
```
`LookupState` 的 `ready` 分支同步加 `relatedPhrases` / `usageNote`，在 `setLookupState` 时透传。

#### 2.2 词卡渲染「相关短语」分区

- 仅在 `lookupKind === "word"` 且 `relatedPhrases?.length` 时渲染
- 位置：例句区之后、底部「加入我的词库」按钮区之前
- 每条 = lemma + translationZh + kind badge（badge 复用 `getPhraseKindLabel` + 现有 amber 样式）
- 整条可点击

#### 2.3 点击相关短语 → 叠短语卡

- `LookupCardProps` 新增可选回调 `onRelatedPhraseClick?: (lemma: string, kind: "collocation" | "phrase" | "idiom") => void`
- 词卡内点击某条相关短语 → 调 `onRelatedPhraseClick(lemma, kind)`
- 由父级（管理 `LookupCardStack` 的组件）push 一张 `{ lookupKind: "phrase", phraseKind: kind, form: lemma }` 卡进 stack
- **复用 PHRASE-001 已有的 2 层上限逻辑**：已是 2 层时拒绝新增（沿用现有提示行为，不另造）
- 找到接入 `LookupCardStack` 的所有页面（`/lectura` `/watch` `/dissect` `/grammar`），把 `onRelatedPhraseClick` 接到现有的 push-to-stack 逻辑上——和 PHRASE-001 里 `onExampleWordClick` 走同一套 push 路径

#### 2.4 construction `usageNote` 展示

- `ready` 且 `usageNote` 非空时，在释义 `<ol>` 之后渲染一个轻量用法说明块（按段 1 设计稿样式）
- 对 word/phrase 卡都适用（construction 词走 word lookup 路径返回 usageNote）
