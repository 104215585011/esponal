# WEB-009 — 设计系统统一 + 首页 Hero + 顶栏导航补全

**优先级**：P1 | **负责人**：Codex1 | **日期**：2026-05-16
**前置依赖**：无
**触发**：Claude2 全站 UI/UX 评审（2026-05-16）发现"两套割裂的视觉语言 + 首页对未登录用户毫无价值主张 + 核心入口缺失"

---

## 背景

Claude2 评审摘录：
> 站内有两套完全不同的视觉语言——首页/搜索/语法是「克制极简版」，learn/auth 是「高饱和大渐变卡片版」。同一个 SiteHeader 串起两套，违和明显。颜色（green vs emerald）、圆角（6/8/10/12/14/20/22/24/28 全用过）、阴影、背景色完全没有 design token。
>
> 首页对未登录新用户毫无价值主张，直接是 YouTube 视频墙，不知道这是学习平台还是镜像站。
>
> 顶栏导航缺少「词库」「视频」入口，词库藏在头像下拉菜单里，新用户找不到。

本 ticket 一次性把"地基"补好。后续所有 UI 改动都基于这套地基做。

## 一、Design Token 统一（tailwind.config.ts）

### 1. 颜色

把全站的 `green-*` 和 `emerald-*` 统一为一组 `brand-*`：

```ts
// tailwind.config.ts
theme: {
  extend: {
    colors: {
      brand: {
        50:  "#ecfdf5",   // emerald-50
        100: "#d1fae5",
        200: "#a7f3d0",
        300: "#6ee7b7",
        400: "#34d399",
        500: "#10b981",   // emerald-500（主交互色）
        600: "#059669",   // emerald-600（按钮 hover、链接）
        700: "#047857",
        800: "#065f46",
        900: "#064e3b"
      }
    }
  }
}
```

**全站替换**：所有 `green-*` 和 `emerald-*` → `brand-*`。
- 用 ripgrep 列出所有引用：`rg -n "green-[0-9]|emerald-[0-9]" src/`
- 用脚本一次性替换：`green-600` / `emerald-600` → `brand-600` 等同色阶
- 注意：`bg-green-50` 和 `bg-emerald-50` 都是 brand-50

### 2. 圆角

定义三档语义化圆角，废弃所有 `rounded-[Npx]` 写法：

```ts
borderRadius: {
  card: "12px",      // 普通卡片、按钮
  surface: "16px",   // 中等容器
  hero: "24px"       // 大型 hero、登录卡
}
```

**替换原则**：
- 列表项、按钮、输入框 → `rounded-card`
- 中等卡片（视频卡、单词卡）→ `rounded-surface`
- Hero、登录注册卡 → `rounded-hero`

### 3. 阴影

```ts
boxShadow: {
  card:     "0 1px 2px rgba(0,0,0,0.04), 0 2px 8px rgba(0,0,0,0.06)",
  elevated: "0 4px 12px rgba(0,0,0,0.07), 0 8px 24px rgba(0,0,0,0.08)",
  hero:     "0 4px 6px -1px rgba(0,0,0,0.07), 0 24px 60px -8px rgba(0,0,0,0.12)"
}
```

替换所有 `shadow-[...]` 内联值为 `shadow-card` / `shadow-elevated` / `shadow-hero`。

### 4. 背景色

```ts
backgroundColor: {
  app:     "#F9FAFB",   // 页面底色
  surface: "#FFFFFF",   // 卡片底色
  muted:   "#F3F4F6"    // 次级背景
}
```

废弃 `bg-[#f8faf7]` `bg-[#f4f7f1]` `bg-[#F9FAFB]` 等多个相似灰绿。

## 二、SiteHeader 补全

### 现状（`src/app/components/web/SiteHeader.tsx`）

只有「课程」「语法」两个一级入口，词库藏在头像下拉里。

### 改造

一级导航补全为四个：

```
[ Esponal logo ]   [ 视频 ]  [ 课程 ]  [ 语法 ]  [ 词库 ]      [ 搜索栏 ]      [ 头像 ]
```

- **视频** → `/`（首页频道流）
- **课程** → `/learn`
- **语法** → `/grammar`
- **词库** → `/vocab`（未登录态显示但点击后跳 `/auth/sign-in?callbackUrl=/vocab`）

样式：active 状态用 `text-brand-600 font-semibold` + 底部 `border-b-2 border-brand-500`，inactive 用 `text-gray-600 hover:text-gray-900`。

## 三、首页 Hero（仅未登录态显示）

### 现状（`src/app/page.tsx`）

直接是搜索栏 + 三个频道横向卡片流，新用户不知道这是什么。

### 新增 Hero

在 SiteHeader 下方、搜索栏上方加一个 hero（**条件：用户未登录时才渲染**，登录后折叠掉）：

```
┌────────────────────────────────────────────────────────────┐
│                                                            │
│   用真实的西语视频，慢慢学会一门语言                         │
│                                                            │
│   不用打卡、不刷题。看 YouTube、点词、查义项、                │
│   下次自动认得 — 你需要的只是好奇心。                          │
│                                                            │
│   [ 立即注册 →]    [ 浏览视频 ]    [ 安装 Chrome 插件 ]      │
│                                                            │
│   已有账号？登录                                             │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

样式规范：
- 背景：`bg-gradient-to-br from-brand-50 to-white`
- 圆角：`rounded-hero`
- 阴影：`shadow-card`
- padding：`py-12 px-10` 桌面，`py-8 px-6` 移动
- 标题字号：`text-3xl lg:text-4xl font-bold tracking-tight`
- 副文字号：`text-base text-gray-600 max-w-[560px]`
- 主 CTA「立即注册」：`bg-brand-600 hover:bg-brand-700 text-white rounded-card px-6 py-3`
- 次 CTA「浏览视频」：`border border-gray-200 text-gray-700 hover:border-brand-400`
- 第三 CTA「安装 Chrome 插件」：暂时 disabled 或链接到一个 stub `/extension`（说明插件 + .crx 下载，本 ticket 不实现，留给 EXT-005）

**判断登录态**：在 `page.tsx`（已经是 server component）里 `const session = await getServerSession(authOptions)`，传给一个 `<HomeHero>` 客户端组件。session 为空时渲染 hero。

## 验收标准（Codex2 用）

1. `rg "green-[0-9]|emerald-[0-9]" src/ | wc -l` 输出 0
2. `tailwind.config.ts` 通过 lint，包含 brand color、三档圆角、三档阴影
3. SiteHeader 在桌面端显示四个一级导航
4. 未登录访问 `/` 看到 hero；登录后访问 `/` 不显示 hero
5. Hero 三个 CTA 都可点击，`立即注册` 跳 `/auth/sign-up`，`浏览视频` 平滑滚动到频道流，`安装 Chrome 插件` 跳 `/extension`（即使是 404 stub 也行）
6. `npm test` 通过
7. `npm run build` 通过
8. 桌面端 1280px、笔记本 1024px、手机模拟 375px 三档下首页都不崩
