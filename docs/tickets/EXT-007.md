## 背景

EXT-006 上线后 PM 复盘出两件事：

1. **`EXT_INGEST_TOKEN` 是过度设计**。token 在公开 .crx 里能反编译拿到，只能挡随便发 POST 的脚本小子；真正起作用的是 write-once + 大小/格式校验 + rate limit + 30 天 TTL 四层防御。token 反而增加运维负担（PM 要生成 / 同步 Vercel / 注入 extension 打包）。
2. **Bootstrap 是 GUI 操作苦活**。需要 PM 装插件、登录 YouTube、点开 60 个视频 tab、等 harvest 触发。每次新视频还得重复。

本 ticket 一次性解决两件事：
- **去 token**，让 ingest 路由只靠现有四层防御
- **写个 Playwright 自动化脚本**，复用 INFRA-003 已经装好的 Chromium，自动开 YouTube tab 跑 harvest

## Part B — Playwright 字幕收割自动化脚本

### 新建 `scripts/bootstrap-harvest.mjs`

```bash
# 用法
node scripts/bootstrap-harvest.mjs --channels=all --recent=20
node scripts/bootstrap-harvest.mjs --channel=UCouyFdE9-Lrjo3M_2idKq1A --recent=10
node scripts/bootstrap-harvest.mjs --videos=dQw4w9WgXcQ,abc123xyz
node scripts/bootstrap-harvest.mjs --videos-from-file=videos.txt
```

### 实现逻辑

```js
import { chromium } from "@playwright/test";
import path from "node:path";

const USER_DATA_DIR = path.resolve(".cache/harvest-chrome-profile");
const EXTENSION_PATH = path.resolve("extension/dist");

async function main() {
  // 1. 解析参数：拿到一个 videoIds 数组
  const videoIds = await resolveVideoList(args);

  // 2. 启动持久化 Chrome（带 extension）
  const browser = await chromium.launchPersistentContext(USER_DATA_DIR, {
    headless: false,  // 关键：必须 headed 才能让 extension 跑
    args: [
      `--disable-extensions-except=${EXTENSION_PATH}`,
      `--load-extension=${EXTENSION_PATH}`
    ]
  });

  // 3. 第一次跑时检测：用户登录了吗？
  const page = await browser.newPage();
  await page.goto("https://www.youtube.com/");
  const loggedIn = await page.locator("button[aria-label*='Account menu']").isVisible({ timeout: 5000 }).catch(() => false);
  if (!loggedIn) {
    console.log("⚠️  Please log into YouTube manually in the opened browser window.");
    console.log("    The script will wait until you've logged in...");
    await page.waitForSelector("button[aria-label*='Account menu']", { timeout: 0 });
    console.log("✓ Logged in. Session saved to profile, you won't need to log in again.");
  }

  // 4. 遍历视频
  console.log(`Harvesting ${videoIds.length} videos...`);
  for (const [i, videoId] of videoIds.entries()) {
    const url = `https://www.youtube.com/watch?v=${videoId}`;
    try {
      await page.goto(url, { waitUntil: "domcontentloaded", timeout: 30_000 });
      // 等 3 秒让 harvest content script 跑完
      await page.waitForTimeout(3_000);
      console.log(`[${i + 1}/${videoIds.length}] ${videoId} ✓`);
    } catch (err) {
      console.log(`[${i + 1}/${videoIds.length}] ${videoId} ✗ ${err.message}`);
    }
  }

  // 5. 给 harvest 最后几个请求一些时间到达后端
  await page.waitForTimeout(5_000);
  await browser.close();
  console.log("Done.");
}

async function resolveVideoList(args) {
  // 三种模式：
  // --channels=all  → 读 src/lib/channels.ts 全部，每个跑 --recent 个
  // --channel=ID    → 指定一个
  // --videos=a,b,c  → 显式
  // --videos-from-file=path → 一行一个
}
```

### 视频发现

复用 `src/lib/youtube.ts` 里现有的 `getChannelUploads()` 函数（如果存在）。如果没有，调用我们自己的 `/api/youtube/channel?id=...` 路由（dev server 要先起着）。

**简化方案**：直接用 yt-dlp 列频道视频 ID（这部分实测**还能用**，只有具体视频的 player API 被拦）：

```bash
python -m yt_dlp --flat-playlist --print "%(id)s" \
  "https://www.youtube.com/channel/{channelId}/videos" | head -n {recent}
```

但增加 yt-dlp + python 依赖，不优雅。建议优先用我们自己 API。

### 验证收割成功

脚本跑完后**不强求**自动校验 Redis（避免还要 import Redis client）——而是打印一条命令让用户验：

```
✓ Done. Verify with:
    redis-cli -u "$REDIS_URL" --scan --pattern "subtitle:*" | head -n 10
```

### 持久化 profile 注意

- `USER_DATA_DIR = .cache/harvest-chrome-profile`，**入 .gitignore**（包含 cookies）
- 第一次跑时 Chrome 弹出来，用户登录 YouTube
- 之后跑都自动复用 session，不需要再登
- 如果 session 过期了（cookie 失效），脚本会再次提示登录

## 五、不在本 ticket 范围内

- **headless harvest**：Chrome extension 在真正 headless 模式下不会运行 content script（这是 Chromium 限制），所以脚本必须 headed。可考虑用 `xvfb`（Linux）伪装显示，但 Windows / Mac 没必要，本 ticket 不做
- **多账号轮换**：单个 YouTube 账号短期高频访问也可能触发 YouTube 限速。如果遇到，加 `--delay-ms=2000` 选项，本 ticket 默认 3s 视频间隔够用
- **CI 跑 harvest**：harvest 是 PM 手动 bootstrap 行为，不要进 CI（消耗时间、需要登录态）
- **服务端调度 harvest**：长远可以做「Vercel cron 触发用户浏览器后台 harvest」，但当前 scope 不做

---

## 六、注意事项

- **`.cache/harvest-chrome-profile/` 入 .gitignore 是硬要求** —— 里面有你的 YouTube cookies，泄露 = 账号被盗
- 第一次跑会下载 ~150MB 的 Chromium（如果 INFRA-003 还没 `npx playwright install chromium` 过），需联网
- harvest 视频 3 秒间隔在 60 个视频的场景下约 3-4 分钟跑完
- 脚本输出每条 `[i/N] videoId ✓/✗`，跑完汇总成功数 / 失败数 / 总耗时
- 失败 videoId 写到 `.cache/harvest-failures.log`，下次可以 `--videos-from-file=.cache/harvest-failures.log` 重试
